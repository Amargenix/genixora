    Robot Kinematics

    Fundamentals of Robot Kinematics

    Kinematics Definition

    Kinematics is the study of motion without considering forces. In robotics, it describes the relationship between:

    • Joint positions (configuration space)
    • End-effector position/orientation (task space)
    • Velocities in both spaces

    Key Concepts

    • Degrees of Freedom (DoF): Number of independent movements
    • Workspace: Reachable space of end-effector
    • Joint Types: Revolute (rotational) vs Prismatic (linear)
    • Configuration Space: All possible joint combinations

    Types of Kinematics Problems

    Forward Kinematics

    Compute end-effector position/orientation from joint angles

    x = f(θ)

    Inverse Kinematics

    Compute joint angles to achieve desired end-effector pose

    θ = f⁻¹(x)

    Mathematical Foundations

    Denavit-Hartenberg (D-H) Convention

    The standard method for assigning coordinate frames to robotic manipulators:

    1. Assign Z-axis along joint axis
    2. X-axis perpendicular to both Z-axes
    3. Y-axis completes right-handed frame

    D-H Parameters Table:

    Joint α (twist) a (length) d (offset) θ (angle)
    1 α₁ a₁ d₁ θ₁
    2 α₂ a₂ d₂ θ₂

    Homogeneous Transformation Matrices

    Represent position and orientation in 4×4 matrix:

    T = [ R p ]
        [ 0 1 ]

    Where R is 3×3 rotation matrix and p is 3×1 position vector

    Forward kinematics as matrix multiplication:

    Tₙ⁰ = T₁⁰ × T₂¹ × ... × Tₙⁿ⁻¹

    Where Tₙ⁰ transforms from joint n to base frame

    Inverse Kinematics Methods

    Analytical Solutions

    Closed-form solutions for specific manipulator geometries:

    • Pieper's Solution: For 6-DoF arms where last 3 axes intersect
    • Geometric Decomposition: Break down into subproblems
    • Algebraic Methods: Solve trigonometric equations
    # Example: 2R planar manipulator IK
    import numpy as np
    def inverse_kinematics(x, y, l1, l2):
      D = (x**2 + y**2 - l1**2 - l2**2)/(2*l1*l2)
      theta2 = np.arctan2(np.sqrt(1-D**2), D)
      theta1 = np.arctan2(y, x) - np.arctan2(l2*np.sin(theta2), l1+l2*np.cos(theta2))
      return theta1, theta2

    Numerical Solutions

    Iterative approaches for general manipulators:

    • Jacobian Methods: Use Jacobian pseudoinverse or transpose
    • Optimization: Formulate as optimization problem
    • Cyclic Coordinate Descent: Iteratively adjust each joint
    # Jacobian Transpose Method
    theta = initial_guess
    for i in range(max_iterations):
      error = desired_pose - current_pose(theta)
      if norm(error) < tolerance:
        break
      J = compute_jacobian(theta)
      delta_theta = alpha * J.T @ error
      theta = theta + delta_theta

    IK Challenges

    Multiple Solutions

    Robots often have multiple IK solutions (elbow up/down configurations)

    Singularities

    Configurations where Jacobian loses rank, causing infinite joint velocities

    No Solution

    When target is outside workspace or physically unreachable

    Differential Kinematics

    Jacobian Matrix

    Relates joint velocities to end-effector velocity:

    ẋ = J(q)q̇

    Where:

    • ẋ is end-effector velocity (6×1)
    • J(q) is Jacobian matrix (6×n)
    • q̇ is joint velocity vector (n×1)

    Velocity Kinematics

    Forward Velocity Kinematics

    Compute end-effector velocity from joint velocities:

    ẋ = J(q)q̇

    Inverse Velocity Kinematics

    Compute joint velocities from end-effector velocity:

    q̇ = J⁺(q)ẋ

    Where J⁺ is pseudoinverse of J

    Singularity Analysis

    Singularities occur when Jacobian loses rank:

    • Workspace Boundary: Arm fully extended or retracted
    • Workspace Interior: Alignment of joint axes

    Detection: Compute determinant or condition number of J·Jᵀ

    Avoidance: Damped least squares, task prioritization

    Example: UR5 Forward Kinematics

                    import numpy as np
                from scipy.spatial.transform import Rotation as R
                
                def dh_transform(theta, d, a, alpha):
                    """Create D-H transformation matrix"""
                    ct = np.cos(theta)
                    st = np.sin(theta)
                    ca = np.cos(alpha)
                    sa = np.sin(alpha)
                    
                    return np.array([
                        [ct, -st*ca,  st*sa, a*ct],
                        [st,  ct*ca, -ct*sa, a*st],
                        [0,     sa,     ca,    d],
                        [0,     0,      0,     1]
                    ])
                
                # UR5 D-H parameters (standard configuration)
                dh_params = [
                    {'theta': 0, 'd': 0.089159, 'a': 0,       'alpha': np.pi/2},
                    {'theta': 0, 'd': 0,         'a': -0.425,  'alpha': 0},
                    {'theta': 0, 'd': 0,         'a': -0.39225,'alpha': 0},
                    {'theta': 0, 'd': 0.10915,   'a': 0,       'alpha': np.pi/2},
                    {'theta': 0, 'd': 0.09465,    'a': 0,       'alpha': -np.pi/2},
                    {'theta': 0, 'd': 0.0823,     'a': 0,       'alpha': 0}
                ]
                
                def ur5_forward_kinematics(joint_angles):
                    """Compute end-effector pose for UR5"""
                    T = np.eye(4)
                    for i in range(6):
                        params = dh_params[i]
                        T_i = dh_transform(joint_angles[i] + params['theta'],
                                          params['d'], params['a'], params['alpha'])
                        T = T @ T_i
                    return T
