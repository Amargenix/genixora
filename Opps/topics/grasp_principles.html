    **General Responsibility Assignment Software Patterns (GRASP)** are **design principles** used in **Object-Oriented Programming (OOP)** to improve **software modularity, maintainability, and flexibility**.

    üìå Why Use GRASP Principles?

    GRASP principles provide guidelines for **assigning responsibilities** in software design.

    • ‚úÖ **Promotes Reusability** ‚Äì Encourages modular and reusable components.

    üí° Key GRASP Principles

    GRASP includes the following core principles:

    • 1. Information Expert ‚Äì Assign responsibility to the class with the most relevant data.
    • 2. Creator ‚Äì Assign the responsibility of creating an object to a class that closely uses it.
    • 3. Controller ‚Äì Assign responsibility for handling user input to a controller class.
    • 4. Low Coupling ‚Äì Minimize dependencies between classes to improve flexibility.
    • 5. High Cohesion ‚Äì Ensure each class has a single, well-defined purpose.
    • 6. Polymorphism ‚Äì Use interfaces or abstract classes to allow multiple implementations.
    • 7. Pure Fabrication ‚Äì Introduce artificial classes to reduce direct dependencies.
    • 8. Indirection ‚Äì Use intermediate objects to manage communication between classes.
    • 9. Protected Variations ‚Äì Use encapsulation and design patterns to shield against changes.
