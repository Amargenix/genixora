    üîÑ Bellman-Ford Algorithm ‚Äì Shortest Path in Graphs

    The **Bellman-Ford Algorithm** is a **graph algorithm** used for finding the **shortest path from a single source vertex to all other vertices** in a **weighted graph**. Unlike **Dijkstra's Algorithm**, Bellman-Ford can **handle graphs with negative weight edges**.

    ‚ö° Key Features of Bellman-Ford Algorithm

    • ‚úÖ Works for both **positive and negative** edge weights.
    • ‚úÖ Detects **negative weight cycles** in the graph.
    • ‚úÖ Solves **Single-Source Shortest Path** problems.
    • ‚ùå **Slower than Dijkstra's Algorithm** (O(V √ó E) time complexity).

    ‚öôÔ∏è How Does the Bellman-Ford Algorithm Work?

    The algorithm works by **iteratively relaxing all edges** in the graph (V - 1) times to guarantee the shortest distance to each vertex.

    üìå Step-by-Step Process

    1. ‚úÖ **Initialize** the distance to all vertices as **infinity (‚àû)**, except the source vertex (0).
    2. üîÑ **Relax all edges** V-1 times (where V = number of vertices).
    3. üö® **Check for Negative Weight Cycles** ‚Äì If relaxation is still possible, a **negative cycle exists**.

    üî¢ Mathematical Formula for Edge Relaxation

    The shortest path estimate is updated using:

    üõ†Ô∏è Bellman-Ford Algorithm Implementation (C++)

    
                #include <iostream>
                #include <vector>
                #include <limits.h>
            
                using namespace std;
            
                struct Edge {
                    int src, dest, weight;
                };
            
                void bellmanFord(vector<Edge> &edges, int V, int E, int source) {
                    vector<int> dist(V, INT_MAX);
                    dist[source] = 0;
            
                    for (int i = 1; i < V; i++) {
                        for (int j = 0; j < E; j++) {
                            int u = edges[j].src;
                            int v = edges[j].dest;
                            int weight = edges[j].weight;
                            if (dist[u] != INT_MAX && dist[u] + weight < dist[v]) {
                                dist[v] = dist[u] + weight;
                            }
                        }
                    }
            
                    // Checking for Negative Weight Cycles
                    for (int j = 0; j < E; j++) {
                        int u = edges[j].src;
                        int v = edges[j].dest;
                        int weight = edges[j].weight;
                        if (dist[u] != INT_MAX && dist[u] + weight < dist[v]) {
                            cout << "Graph contains a negative weight cycle!";
                            return;
                        }
                    }
            
                    // Printing the shortest distances
                    cout << "Vertex Distance from Source:\n";
                    for (int i = 0; i < V; i++) {
                        cout << i << " \t\t " << dist[i] << endl;
                    }
                }
            
                int main() {
                    int V = 5; // Number of vertices
                    int E = 8; // Number of edges
                    vector<Edge> edges = {
                        {0, 1, -1}, {0, 2, 4}, {1, 2, 3}, {1, 3, 2},
                        {1, 4, 2}, {3, 2, 5}, {3, 1, 1}, {4, 3, -3}
                    };
            
                    bellmanFord(edges, V, E, 0);
                    return 0;
                }

    üîÑ Comparison: Bellman-Ford vs Dijkstra's Algorithm

    Feature Bellman-Ford Dijkstra's Algorithm
    Time Complexity O(V √ó E) O(V log V) (using Priority Queue)
    Handles Negative Edges ‚úÖ Yes ‚ùå No
    Handles Negative Cycles ‚úÖ Yes ‚ùå No
    Performance Slower (Iterative) Faster (Greedy Approach)
