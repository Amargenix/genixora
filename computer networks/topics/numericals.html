    Numerical Concepts in Computer Networks

    Introduction to Network Numerics

    Numerical concepts form the foundation for understanding network performance, capacity planning, and troubleshooting. These calculations help network engineers design efficient systems and diagnose problems.

    1. Bandwidth and Throughput Calculations

    Theoretical Maximum Throughput

    Calculate the maximum data transfer rate for a channel:

    Throughput (bps) = Bandwidth × log₂(1 + SNR)

    Where SNR is Signal-to-Noise Ratio

    Example: For 10 MHz bandwidth with SNR of 30 dB:
    Throughput ≈ 10 MHz × log₂(1001) ≈ 99.7 Mbps

    Effective Throughput

    Accounting for protocol overhead:

    Effective Throughput = (Payload Size / Total Packet Size) × Link Rate
    Example: 1500B payload with 100B overhead on 1 Gbps link:
    Effective Throughput = (1500/1600) × 1 Gbps = 937.5 Mbps

    Bandwidth-Delay Product

    Measures network pipe capacity:

    BDP (bits) = Bandwidth (bps) × RTT (seconds)
    Example: 100 Mbps link with 50ms RTT:
    BDP = 100 × 10⁶ × 0.05 = 5,000,000 bits (625 KB)

    2. Latency Calculations

    Total Network Latency

    Total Latency = Propagation + Transmission + Queuing + Processing

    Components:

    • Propagation: Distance/speed of light
    • Transmission: Packet size/bandwidth
    • Queuing: Router buffer delays
    • Processing: Device handling time

    Propagation Delay

    Propagation Delay = Distance / Speed of Light in Medium

    Speed of light in fiber ≈ 2 × 10⁸ m/s

    Example: 1000 km fiber link:
    Delay = 1,000,000 m / 2 × 10⁸ m/s = 5 ms

    Transmission Delay

    Transmission Delay = Packet Size (bits) / Bandwidth (bps)
    Example: 1500B packet on 100 Mbps link:
    Delay = (1500 × 8) / 100 × 10⁶ = 0.12 ms

    3. IP Addressing Calculations

    Subnetting Calculations

    Number of Subnets = 2n
    Hosts per Subnet = 2(32-n) - 2

    Where n = number of borrowed bits

    Example: 192.168.1.0/24 with 3 borrowed bits:
    Subnets = 2³ = 8
    Hosts/subnet = 2⁵ - 2 = 30

    CIDR Block Sizing

    Block Size = 2(32 - prefix)
    Example: /26 prefix:
    Block Size = 2⁶ = 64 addresses

    IPv6 Addressing

    Available Addresses = 2(128 - n)

    Where n = prefix length

    Example: /64 subnet:
    Addresses = 2⁶⁴ ≈ 1.8 × 10¹⁹

    4. Error Detection and Correction

    Checksum Calculation

    Simple error detection method:

    1. Divide data into 16-bit words
    2. Sum all words
    3. Add carry to result
    4. Take 1's complement
    Example: For words 0x4500 and 0x0034:
    Sum = 0x4500 + 0x0034 = 0x4534
    Checksum = ~0x4534 = 0xBACB

    CRC Calculation

    Cyclic Redundancy Check steps:

    1. Append n zeros to data
    2. Divide by generator polynomial
    3. Remainder is CRC

    Common polynomials: CRC-32 (Ethernet), CRC-16 (USB)

    Hamming Code

    Error correction code calculation:

    Parity Bits = 2p ≥ p + data bits + 1
    Example: For 4 data bits:
    2³ ≥ 3 + 4 + 1 → 8 ≥ 8
    So 3 parity bits needed

    5. Network Performance Metrics

    Metric Formula Description
    Utilization (Throughput / Capacity) × 100% Percentage of capacity used
    Goodput (Useful Data / Total Data) × Throughput Application-level throughput
    Packet Loss (Lost Packets / Sent Packets) × 100% Percentage of packets not received
    Jitter σ(Delayi) Standard deviation of packet delays

    Practical Numerical Examples

    File Transfer Time

    Time = File Size / Effective Throughput
    Example: 500 MB file over 80 Mbps connection:
    Time = (500 × 8 × 10⁶) / (80 × 10⁶) = 50 seconds

    VoIP Bandwidth

    Bandwidth = (Packet Size × 8 × Packets/Second) / 1000 kbps
    Example: G.711 codec (50 pps, 200B packets):
    BW = (200 × 8 × 50) / 1000 = 80 kbps per call

    Network Capacity

    Users = (Total BW × Efficiency) / BW per User
    Example: 100 Mbps LAN with 70% efficiency for 5 Mbps users:
    Users = (100 × 0.7) / 5 = 14 simultaneous users
