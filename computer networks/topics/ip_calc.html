    Subnetting Calculations

    Basic Subnetting Formulas

    Number of subnets: 2n (where n = borrowed bits)

    Hosts per subnet: 2h - 2 (where h = host bits)

    Block size: 256 - subnet mask octet value

    Subnet mask: /n (CIDR) or dotted decimal (e.g., 255.255.255.0)

    VLSM (Variable Length Subnet Masking)

    VLSM Concepts

    • Allows different subnet masks in the same network
    • More efficient use of address space
    • Requires careful planning to avoid overlaps
    • Common in real-world networks with varying subnet sizes

    VLSM Calculation Steps

    1. List all needed subnets with host requirements
    2. Sort subnets from largest to smallest
    3. Allocate largest subnets first from the main block
    4. Continue allocating smaller subnets from remaining space
    5. Document all allocations carefully

    VLSM Example

    Given: 192.168.1.0/24 network with requirements:

    • Subnet A: 60 hosts
    • Subnet B: 30 hosts
    • Subnet C: 12 hosts
    • Subnet D: 5 hosts

    Solution:

    1. Subnet A: 192.168.1.0/26 (62 hosts)
    2. Subnet B: 192.168.1.64/27 (30 hosts)
    3. Subnet C: 192.168.1.96/28 (14 hosts)
    4. Subnet D: 192.168.1.112/29 (6 hosts)

    Remaining: 192.168.1.120/29 to 192.168.1.255

    IPv6 Calculations

    IPv6 Subnetting

    IPv6 uses a fixed /64 for most subnets, providing 18 quintillion hosts per subnet.

    Global Routing Prefix: Typically /48 (assigned by ISP)

    Subnet ID: 16 bits (allowing 65,536 /64 subnets)

    Interface ID: 64 bits (EUI-64 or random)

    IPv6 Address Planning

    Example Allocation:

    2001:db8:acad::/48

    • Branch 1: 2001:db8:acad:1::/64
    • Branch 2: 2001:db8:acad:2::/64
    • WAN Links: 2001:db8:acad:ff00::/56

    IPv6 Address Shortening

    Rules for compressing IPv6 addresses:

    1. Leading zeros in each hextet can be omitted
    2. One sequence of consecutive all-zero hextets can be replaced with ::
    3. Double colon can only be used once in an address

    Example:

    Full: 2001:0db8:0000:0000:0000:ff00:0042:8329

    Compressed: 2001:db8::ff00:42:8329

    IPv6 EUI-64 Calculation

    How to derive an IPv6 interface ID from a MAC address:

    MAC: 00:1A:2B:3C:4D:5E

    1. Split MAC: 00-1A-2B and 3C-4D-5E
    2. Insert FF-FE in middle: 00-1A-2B-FF-FE-3C-4D-5E
    3. Flip 7th bit: 02-1A-2B-FF-FE-3C-4D-5E
    4. Resulting Interface ID: 021a:2bff:fe3c:4d5e

    Network Range Calculations

    Finding Network Boundaries

    To calculate network address and broadcast address:

    Given: 172.16.35.123/20

    1. Mask in binary: 255.255.240.0
    2. Block size: 256 - 240 = 16
    3. Find multiple: floor(35/16) √ó 16 = 32
    4. Network: 172.16.32.0
    5. Broadcast: 172.16.47.255
    6. Range: 172.16.32.1 - 172.16.47.254

    Supernetting (Route Summarization)

    Combining multiple networks into a single summarized route:

    Given networks:

    • 192.168.0.0/24
    • 192.168.1.0/24
    • 192.168.2.0/24
    • 192.168.3.0/24

    Steps:

    1. Convert to binary: 11000000.10101000.00000000.00000000
    2. Find common prefix: 22 bits
    3. Summary route: 192.168.0.0/22
