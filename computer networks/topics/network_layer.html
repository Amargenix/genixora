Computer Networks

    Network Layer (Layer 3)

    Introduction to Network Layer

    The network layer is the third layer in the OSI model and is responsible for packet forwarding including routing through intermediate routers. It provides the functional and procedural means of transferring variable-length data sequences from a source to a destination host via one or more networks.

    Key Functions of Network Layer

    Routing

    Determines the optimal path for packet delivery from source to destination using routing algorithms like OSPF, BGP, RIP.

    • Static vs Dynamic routing
    • Distance-vector vs Link-state protocols
    • Path determination

    Logical Addressing

    Provides unique addressing system (IP addresses) to identify devices across networks.

    • IPv4 (32-bit) and IPv6 (128-bit)
    • Public vs Private IPs
    • Subnetting and Supernetting

    Packet Forwarding

    Moves packets from a router's input port to appropriate output port.

    • Destination-based forwarding
    • Generalized forwarding
    • Longest prefix matching

    Fragmentation & Reassembly

    Breaks large packets into smaller ones when crossing networks with smaller MTUs.

    • MTU (Maximum Transmission Unit)
    • Path MTU Discovery
    • Reassembly at destination

    Network Layer Protocols

    Internet Protocol (IP)

    The principal communications protocol for relaying datagrams across network boundaries.

    IPv4

    • 32-bit address space (4.3 billion addresses)
    • Connectionless, unreliable protocol
    • Header includes TTL, protocol, checksum
    • Supports fragmentation

    IPv6

    • 128-bit address space (3.4×10³⁸ addresses)
    • Simplified header format
    • Built-in security (IPsec)
    • No fragmentation support

    Routing Algorithms

    Distance Vector

    • Routers share entire routing tables with neighbors
    • Uses Bellman-Ford algorithm
    • Examples: RIP, IGRP
    • Count-to-infinity problem
    • Slow convergence

    Link State

    • Routers share info about directly connected links
    • Uses Dijkstra's algorithm
    • Examples: OSPF, IS-IS
    • Faster convergence
    • Requires more memory/CPU

    Path Vector

    • Records complete path information
    • Used for inter-domain routing
    • Example: BGP
    • Policy-based routing
    • Scalable for large networks

    Network Layer Services

    Connectionless Service (Datagram)

    • No path establishment phase
    • Each packet routed independently
    • Packets may arrive out of order
    • Used by IP protocol
    • More robust to failures

    Connection-Oriented Service (Virtual Circuit)

    • Path established before data transfer
    • All packets follow same path
    • In-sequence packet delivery
    • Used by ATM, X.25
    • QoS guarantees possible

    Network Layer in TCP/IP Model

    The TCP/IP model combines the OSI's physical and data link layers into a single Network Interface layer, and the session, presentation, and application layers into a single Application layer. The Network Layer remains largely unchanged.

    Application Layer
    Transport Layer (TCP/UDP)
    Internet Layer (IP)
    Network Access Layer

    The Internet Layer in TCP/IP is equivalent to the Network Layer in OSI, responsible for host-to-host communication and routing packets across networks.

    Real-World Applications

    Internet Routing

    BGP routing between autonomous systems forms the backbone of the internet.

    VPNs

    Virtual Private Networks use tunneling protocols that operate at the network layer.

    NAT

    Network Address Translation allows multiple devices to share a single public IP.

    Quality of Service

    Network layer QoS mechanisms prioritize certain types of traffic.
