Computer Networks

    Network Topologies

    Introduction to Network Topologies

    A network topology refers to the physical or logical arrangement of nodes (computers, switches, routers, etc.) and connections in a network. It defines how different nodes communicate with each other and how data flows through the network.

    Choosing the right topology is crucial for network performance, reliability, and scalability.

    Key Considerations

    • Cost: Cable/equipment requirements
    • Scalability: Ease of adding new nodes
    • Reliability: Fault tolerance
    • Performance: Data transfer speed
    • Management: Ease of troubleshooting

    Physical Topologies

    Bus Topology

    All devices connected to a single backbone cable (trunk) with terminators at both ends.

    Characteristics:

    • Simple and inexpensive
    • Single point of failure (backbone)
    • Collisions occur (CSMA/CD)
    • Difficult to troubleshoot

    Example: Early Ethernet networks (10Base2, 10Base5)

    Star Topology

    All devices connected to a central hub or switch.

    Characteristics:

    • Easy to install and manage
    • Central point of failure (hub/switch)
    • No collisions (with switch)
    • Scalable - easy to add nodes

    Example: Modern Ethernet networks, WiFi networks

    Ring Topology

    Devices connected in a circular fashion with data traveling in one direction.

    Characteristics:

    • Equal network access for all nodes
    • Single point of failure (unless dual-ring)
    • Token passing mechanism
    • Difficult to add/remove nodes

    Example: Token Ring (IEEE 802.5), FDDI

    Mesh Topology

    Every device connected to every other device in the network.

    Characteristics:

    • Highly fault tolerant
    • Redundant paths
    • Expensive to implement
    • Complex to manage

    Types: Full mesh (all connected), Partial mesh (some connected)

    Example: WANs, Internet backbone

    Tree Topology

    Combination of star topologies connected via bus topology (hierarchical).

    Characteristics:

    • Scalable for large networks
    • Point-to-point wiring
    • Root node failure affects entire network
    • Used in wide area networks

    Example: Enterprise networks, cable TV networks

    Hybrid Topology

    Combination of two or more different topologies.

    Characteristics:

    • Flexible and reliable
    • Can optimize strengths of different topologies
    • Complex design
    • Expensive to implement

    Example: Star-Ring, Star-Bus networks

    Physical Topology Comparison

    Topology Cost Scalability Fault Tolerance Complexity
    Bus Low Poor Low Low
    Star Medium Good Medium Low
    Ring Medium Fair Low (unless dual-ring) Medium
    Mesh High Excellent High High
    Tree Medium-High Good Medium Medium
    Hybrid High Excellent High High

    Logical Topologies

    Logical Bus

    All devices receive all transmissions but only the intended recipient processes it.

    Characteristics:

    • Signals broadcast to all devices
    • Used in Ethernet networks
    • MAC addresses determine recipients
    • CSMA/CD access method

    Logical Ring

    Data travels sequentially from one device to another until it reaches its destination.

    Characteristics:

    • Token passing mechanism
    • Deterministic access
    • Used in Token Ring and FDDI
    • No collisions

    Logical Star

    Central device controls all communications (even if physically wired differently).

    Characteristics:

    • Switch makes forwarding decisions
    • Point-to-point connections
    • Full-duplex communication
    • Common in modern networks

    Physical vs Logical Topology

    Aspect Physical Topology Logical Topology
    Definition Actual layout of devices and cables How data flows between nodes
    Visibility Can be seen and touched Conceptual, not visible
    Example Star wiring with cables Ethernet's logical bus
    Change Impact Requires rewiring Changed via configuration
    Performance Affects signal quality Affects data flow efficiency

    Wireless Topologies

    Infrastructure Mode

    Devices communicate through an access point (central device).

    Characteristics:

    • Similar to star topology
    • Centralized management
    • Better security
    • Extended service set (ESS) for roaming

    Example: Enterprise WiFi networks

    Ad-hoc Mode

    Devices communicate directly with each other without an access point.

    Characteristics:

    • Decentralized
    • Quick to set up
    • Limited range
    • Less secure

    Example: Device-to-device file sharing

    Mesh Networks

    Wireless devices form a mesh with multiple connection paths.

    Characteristics:

    • Self-healing
    • Self-configuring
    • Highly reliable
    • Complex to manage

    Example: Smart home systems, municipal WiFi

    Topology Selection Criteria

    Network Size

    • Small networks: Star or bus
    • Medium networks: Star or tree
    • Large networks: Tree or hybrid
    • Enterprise networks: Hierarchical (core, distribution, access)

    Budget Constraints

    • Low budget: Bus or simple star
    • Medium budget: Star with switches
    • High budget: Mesh or hybrid
    • Consider both initial and maintenance costs

    Performance Needs

    • High bandwidth: Star or mesh
    • Low latency: Star or point-to-point
    • Deterministic performance: Ring
    • Scalability: Tree or hybrid

    Reliability Requirements

    • Mission-critical: Mesh or dual-ring
    • Fault tolerance: Redundant paths
    • High availability: Multiple connections
    • Single points of failure: Avoid bus or simple star

    Emerging Topology Trends

    Software-Defined Networking

    Decoupling control plane from data plane, enabling dynamic topology changes.

    Intent-Based Networking

    Networks that automatically adjust topology based on business policies.

    Edge Computing

    Distributed topology with processing at the edge of the network.

    5G Network Slicing

    Virtual topologies created on demand for specific applications.
