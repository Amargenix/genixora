Computer Networks

    Local Area Network (LAN)

    Introduction to LAN

    A Local Area Network (LAN) is a computer network that interconnects computers within a limited area such as a residence, school, laboratory, university campus, or office building.

    LANs are characterized by higher data transfer rates, smaller geographic range, and lack of need for leased telecommunication lines.

    Key Characteristics

    • High data transfer rates (10 Mbps to 100 Gbps)
    • Limited geographical area (typically < 1 km)
    • Full-time connectivity to local services
    • Typically owned by the organization using it
    • Lower latency compared to WANs

    LAN Topologies

    Bus Topology

    All devices connected to a single backbone cable. Simple but prone to collisions.

    Pros: Easy to implement, requires less cable

    Cons: Single point of failure, difficult to troubleshoot

    Star Topology

    All devices connected to a central hub/switch. Most common in modern LANs.

    Pros: Easy to add/remove devices, fault tolerant

    Cons: Requires more cable, hub/switch is single point of failure

    Ring Topology

    Devices connected in a circular fashion, with data traveling in one direction.

    Pros: Equal network access, performs well under heavy load

    Cons: Failure of one device affects entire network

    Mesh Topology

    Devices interconnected with many redundant connections.

    Pros: Highly fault tolerant, multiple paths for data

    Cons: Expensive to implement, complex to manage

    LAN Technologies and Standards

    Ethernet (IEEE 802.3)

    The most widely used LAN technology, defining wiring and signaling standards for the physical layer and MAC.

    Standard Speed Cable Type Max Distance
    10BASE-T 10 Mbps Cat3 UTP 100m
    100BASE-TX 100 Mbps Cat5 UTP 100m
    1000BASE-T 1 Gbps Cat5e UTP 100m
    10GBASE-T 10 Gbps Cat6a UTP 100m
    40GBASE-T 40 Gbps Cat8 UTP 30m

    Wi-Fi (IEEE 802.11)

    Wireless LAN technology allowing devices to connect without physical cables.

    • 802.11a: 5 GHz, 54 Mbps
    • 802.11b: 2.4 GHz, 11 Mbps
    • 802.11g: 2.4 GHz, 54 Mbps
    • 802.11n: 2.4/5 GHz, 600 Mbps
    • 802.11ac: 5 GHz, 1.3 Gbps
    • 802.11ax (Wi-Fi 6): 2.4/5 GHz, 10 Gbps

    Other LAN Technologies

    • Token Ring (IEEE 802.5): Older technology using token passing
    • FDDI: Fiber Distributed Data Interface (100 Mbps over fiber)
    • ARCNET: Attached Resource Computer NETwork (obsolete)
    • Powerline Networking: Uses electrical wiring for data

    LAN Protocols and Addressing

    MAC Addressing

    48-bit hardware address (e.g., 00:1A:2B:3C:4D:5E) burned into network interface.

    • First 24 bits: OUI (Organizationally Unique Identifier)
    • Last 24 bits: Device identifier
    • Used for local delivery (Layer 2)

    ARP (Address Resolution Protocol)

    Resolves IP addresses to MAC addresses for local delivery.

    • Maintains ARP cache
    • ARP request (broadcast)
    • ARP reply (unicast)

    LAN Design Considerations

    Physical Design

    • Cable selection (Cat5e, Cat6, fiber)
    • Topology selection
    • Rack and cabling organization
    • Power requirements
    • Environmental factors

    Logical Design

    • IP addressing scheme
    • VLAN configuration
    • Subnetting strategy
    • Routing protocols
    • Naming conventions

    Performance Factors

    Bandwidth

    Data transfer capacity (e.g., 1 Gbps)

    Throughput

    Actual data transfer rate achieved

    Latency

    Time for data to travel end-to-end

    Jitter

    Variation in packet arrival times
