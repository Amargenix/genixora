    Network Address Translation (NAT)

    Definition: Network Address Translation (NAT) is a method of remapping one IP address space into another by modifying network address information in the IP header of packets while they are in transit across a traffic routing device.

    Why NAT is Needed

    • IPv4 address exhaustion - NAT helps conserve global IP addresses
    • Security - Hides internal network structure from external networks
    • Flexibility - Allows changes in internal IP addressing without affecting external connectivity
    • Cost reduction - Reduces the need for multiple public IP addresses

    Types of NAT

    1. Static NAT

    One-to-one mapping between local and global addresses.

    Example: Mapping a public IP to a web server's private IP.

    2. Dynamic NAT

    Maps private IP addresses to public addresses from a pool.

    Example: A company with 100 private IPs and 50 public IPs in a pool.

    3. PAT (Port Address Translation) or NAT Overload

    Maps multiple private IPs to a single public IP using different ports.

    Example: Home router allowing multiple devices to share one public IP.

    How NAT Works

    The basic NAT process involves:

    1. Internal host sends packet to external destination
    2. NAT device replaces source IP with public IP
    3. NAT device maintains translation table
    4. Response comes back to public IP
    5. NAT device forwards response to correct internal host
    NAT Type IP Mapping Ports Used Common Use
    Static NAT 1-to-1 fixed Same Servers
    Dynamic NAT 1-to-1 from pool Same Medium networks
    PAT/NAT Overload Many-to-1 Different Home/Small office

    Advantages and Disadvantages

    Advantages

    • Address conservation: Saves IPv4 addresses
    • Security: Hides internal network structure
    • Flexibility: Easy to change internal addressing
    • Cost-effective: Reduces need for public IPs

    Disadvantages

    • Performance impact: Adds processing overhead
    • Complexity: Can cause issues with some applications
    • End-to-end connectivity: Breaks true end-to-end connectivity
    • Troubleshooting: Makes network debugging harder

    NAT Terminology

    • Inside Local: Private IP address assigned to host on internal network
    • Inside Global: Public IP address representing inside host to outside world
    • Outside Local: IP address of external host as it appears to internal network
    • Outside Global: Public IP address assigned to host in external network

    NAT Configuration Examples

    Cisco Router NAT Configuration

                    interface GigabitEthernet0/0
                 ip address 192.168.1.1 255.255.255.0
                 ip nat inside
                !
                interface GigabitEthernet0/1
                 ip address 203.0.113.2 255.255.255.0
                 ip nat outside
                !
                ip nat inside source list 1 interface GigabitEthernet0/1 overload
                !
                access-list 1 permit 192.168.1.0 0.0.0.255

    NAT and IPv6

    With IPv6's large address space, NAT is generally not needed for address conservation. However, some forms of NAT are still used in IPv6 for:

    • Privacy extensions
    • Network merging
    • Transition mechanisms between IPv4 and IPv6
