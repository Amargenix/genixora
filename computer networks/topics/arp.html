    ARP (Address Resolution Protocol)

    Address Resolution Protocol (ARP) - Complete Guide

    1. Introduction to ARP

    The Address Resolution Protocol (ARP) is a fundamental protocol used in TCP/IP networks to map IP addresses to their corresponding MAC (Media Access Control) addresses.

    Key Characteristics of ARP

    • Operates at Layer 2 (Data Link Layer) of the OSI model
    • Resolves IPv4 addresses to MAC addresses
    • Works only within the same local network segment
    • Defined in RFC 826 (1982)

    Why ARP is Needed

                    +----------------+       +----------------+
                |                |       |                |
                |   IP Address   | ----> |   MAC Address  |
                |   (Logical)    |       |   (Physical)   |
                |                |       |                |
                +----------------+       +----------------+

    Network devices need both IP and MAC addresses for communication:

    • IP addresses are used for end-to-end delivery across networks
    • MAC addresses are used for hop-to-hop delivery on the same network

    2. ARP Operation Fundamentals

    Basic ARP Process

                    1. Host A wants to send to Host B (IP known, MAC unknown)
                2. Host A broadcasts ARP Request: "Who has IP B? Tell A"
                3. Host B responds with ARP Reply: "IP B is at MAC B"
                4. Host A caches this mapping and sends frames to MAC B

    ARP Packet Types

    Packet Type Opcode Description
    ARP Request 1 Broadcast query for MAC address
    ARP Reply 2 Unicast response with MAC address
    RARP Request 3 Reverse ARP request (obsolete)
    RARP Reply 4 Reverse ARP reply (obsolete)

    ARP Cache

    Devices maintain an ARP cache (table) that stores recent IP-to-MAC address mappings:

                    C:\> arp -a
                
                Interface: 192.168.1.100 --- 0xa
                  Internet Address      Physical Address      Type
                  192.168.1.1           00-1a-2b-3c-4d-5e    dynamic
                  192.168.1.255         ff-ff-ff-ff-ff-ff    static

    Field Descriptions

    Field Size (bytes) Description Example Values
    Hardware Type 2 Type of hardware interface 1 (Ethernet)
    Protocol Type 2 Type of protocol address 0x0800 (IPv4)
    HW Addr Length 1 Length of MAC address 6
    Proto Addr Length 1 Length of protocol address 4
    Operation 2 ARP request or reply 1 (request), 2 (reply)
    Sender HW Addr 6 MAC of sender 00:1A:2B:3C:4D:5E
    Sender Proto Addr 4 IP of sender 192.168.1.100
    Target HW Addr 6 MAC of target 00:00:00:00:00:00 (request)
    Target Proto Addr 4 IP of target 192.168.1.1

    4. ARP Operation in Detail

    ARP Request Example

                    Source MAC: 00:1A:2B:3C:4D:5E (Host A)
                Destination MAC: FF:FF:FF:FF:FF:FF (Broadcast)
                ARP Operation: 1 (Request)
                Sender MAC: 00:1A:2B:3C:4D:5E
                Sender IP: 192.168.1.100
                Target MAC: 00:00:00:00:00:00
                Target IP: 192.168.1.1

    ARP Reply Example

                    Source MAC: 00:1B:2C:3D:4E:5F (Host B)
                Destination MAC: 00:1A:2B:3C:4D:5E (Host A)
                ARP Operation: 2 (Reply)
                Sender MAC: 00:1B:2C:3D:4E:5F
                Sender IP: 192.168.1.1
                Target MAC: 00:1A:2B:3C:4D:5E
                Target IP: 192.168.1.100

    ARP Cache Behavior

    Action Cache Update Timeout
    Receive ARP Reply Add/update entry Typically 10-20 min
    Send ARP Request Create incomplete entry Few seconds (retry timeout)
    Receive ARP Request Update sender's info Same as above

    5. ARP Variants and Related Protocols

    Types of ARP

    Proxy ARP

    • Router answers ARP requests on behalf of hosts
    • Used when hosts are on different physical networks
    • Helps with network renumbering

    Gratuitous ARP

    • Host announces its IP-to-MAC mapping
    • Used for:
      • Duplicate IP detection
      • Failover scenarios
      • Cache updates

    Reverse ARP (RARP)

    • Obsolete protocol
    • Used to find IP from MAC
    • Replaced by DHCP and BOOTP

    Related Protocols

    Protocol Purpose Relation to ARP
    NDP (IPv6) IPv6 neighbor discovery Replaces ARP in IPv6
    Inverse ARP Frame Relay DLCI to IP mapping Similar concept
    DHCP IP address assignment Replaced RARP

    6. ARP in Different Network Scenarios

    ARP Across Subnets

                    +--------+      +--------+      +--------+
                | Host A |------| Router |------| Host B |
                +--------+      +--------+      +--------+
                192.168.1.100  192.168.1.1     10.0.0.100
                               10.0.0.1
                
                1. Host A ARPs for router (192.168.1.1)
                2. Router forwards packet to Host B
                3. Router may ARP for Host B if needed

    ARP in VLAN Environments

    • ARP requests are confined to the originating VLAN
    • Router or Layer 3 switch performs inter-VLAN routing
    • Proxy ARP may be used in some configurations

    ARP in Wireless Networks

    Scenario ARP Behavior
    Infrastructure Mode Normal ARP, AP handles broadcasts
    Ad-hoc Mode Direct ARP between stations
    Mesh Networks May use modified ARP or proxy

    7. ARP Cache Management

    ARP Cache Operations

    Viewing ARP Cache

                    # Windows
                arp -a
                
                # Linux/Unix
                arp -n
                ip neigh show

    Clearing ARP Cache

                    # Windows
                netsh interface ip delete arpcache
                
                # Linux
                ip neigh flush all
                
                # Cisco IOS
                clear arp-cache

    Adding Static Entries

                    # Windows
                arp -s 192.168.1.1 00-1a-2b-3c-4d-5e
                
                # Linux
                arp -i eth0 -s 192.168.1.1 00:1a:2b:3c:4d:5e

    ARP Cache Timeout Values

    System Default Timeout Configurable
    Windows 15-45 seconds (incomplete)
    10-20 minutes (complete)    		 Yes (registry)
    Linux 30 seconds (incomplete)
    60 seconds (reachable)    		 Yes (sysctl)
    Cisco IOS 4 hours Yes (arp timeout)

    8. ARP Security Considerations

    ARP Vulnerabilities

    ARP Spoofing

    • Attacker sends fake ARP replies
    • Can intercept or modify traffic
    • Basis for man-in-the-middle attacks

    ARP Flooding

    • Overwhelms switch CAM tables
    • Can lead to switch fail-open
    • Enables sniffing in switched networks

    ARP Security Mechanisms

    Mechanism Description Effectiveness
    Static ARP Entries Manually configured mappings High (but not scalable)
    ARP Inspection Validates ARP packets High (requires switch support)
    Private VLANs Limits communication between hosts Medium
    Encryption Protects data confidentiality High (but doesn't prevent spoofing)

    ARP Spoofing Detection

    • Monitor for multiple MACs claiming same IP
    • Watch for MAC addresses changing frequently
    • Use tools like Arpwatch or XArp

    9. ARP in IPv6 (NDP)

    Neighbor Discovery Protocol (NDP)

    IPv6 replaces ARP with the Neighbor Discovery Protocol (NDP), which provides additional functionality:

    • Uses ICMPv6 messages instead of ARP
    • Performs address resolution (like ARP)
    • Also handles router discovery, address autoconfiguration
    • More secure with Cryptographically Generated Addresses (CGA)

    NDP vs ARP Comparison

    Feature ARP (IPv4) NDP (IPv6)
    Protocol Separate protocol Part of ICMPv6
    Message Types Request/Reply Neighbor Solicitation/Advertisement
    Security None (vulnerable to spoofing) SEcure Neighbor Discovery (SEND)
    Multicast Use Broadcast Solicited-node multicast

    10. ARP Troubleshooting

    Common ARP Issues

    Duplicate IP Addresses

    • Multiple devices respond to ARP for same IP
    • Detected via gratuitous ARP
    • Causes intermittent connectivity

    ARP Timeouts

    • Entries expire too quickly
    • Causes frequent ARP traffic
    • Adjust timeout values if needed

    Incomplete ARP

    • No response to ARP requests
    • Check physical connectivity
    • Verify target host is on same network

    ARP Troubleshooting Commands

    Task Windows Linux Cisco IOS
    View ARP cache arp -a arp -n show arp
    Clear ARP cache netsh interface ip delete arpcache ip neigh flush all clear arp-cache
    Monitor ARP Network Monitor tcpdump -i eth0 arp debug arp

    11. ARP in Network Devices

    ARP in Switches

    • Switches operate at Layer 2 and don't use ARP themselves
    • Maintain MAC address tables (CAM tables) for forwarding
    • Flood ARP requests to all ports (broadcast)
    • May have ARP inspection features for security

    ARP in Routers

    Function Description
    Proxy ARP Respond to ARP requests for other networks
    ARP for Next Hop Resolve MAC of next router when forwarding
    ARP Cache Maintain IP-to-MAC mappings for connected networks

    ARP in Firewalls

    Stateful Inspection

    Track ARP conversations to detect anomalies

    ARP Filtering

    Block malicious ARP packets

    ARP Spoofing Protection

    Prevent unauthorized ARP updates

    12. Advanced ARP Concepts

    ARP for Different Media Types

    Media Type ARP Adaptation
    Ethernet Standard ARP (RFC 826)
    Token Ring Uses different hardware type
    Frame Relay Uses Inverse ARP
    ATM Uses ATMARP

    ARP in Virtualization

    Virtual Machines

    • Hypervisor may handle ARP for VMs
    • Virtual switches maintain ARP caches
    • Potential for ARP spoofing between VMs

    Containers

    • Host typically handles ARP
    • Bridge networks may have separate ARP tables
    • Overlay networks may use proxy ARP

    ARP in Software Defined Networking

    • SDN controllers may centralize ARP handling
    • OpenFlow switches can proxy ARP requests
    • ARP traffic can be managed via flow rules
    • Potential for ARP optimization in SDN
