    Classless Inter-Domain Routing (CIDR)

    Introduction to CIDR

    CIDR (Classless Inter-Domain Routing) was introduced in 1993 to replace the classful network addressing architecture and provide more efficient IP address allocation. It allows for:

    • More flexible allocation of IP addresses
    • Reduced routing table sizes
    • Slowing the exhaustion of IPv4 addresses
    • Elimination of class boundaries (A, B, C networks)

    CIDR Notation

    CIDR notation represents an IP address and its associated routing prefix in the format:

    192.168.1.0/24

    Where:

    • 192.168.1.0 is the network address
    • /24 indicates the first 24 bits are the network portion
    CIDR Notation Subnet Mask Number of Addresses
    /24 255.255.255.0 256
    /25 255.255.255.128 128
    /26 255.255.255.192 64

    Benefits of CIDR

    Efficient Address Space Utilization

    CIDR allows for allocation of IP addresses in blocks that closely match actual needs, reducing waste.

    Route Aggregation

    Multiple IP networks can be represented as a single routing table entry, reducing the size of routing tables.

    Flexible Network Sizes

    Networks can be any size (power of two) rather than being constrained to class A, B, or C sizes.

    CIDR Calculation Examples

    Example 1: 192.168.1.0/24

    - Network address: 192.168.1.0

    - Broadcast address: 192.168.1.255

    - Usable hosts: 192.168.1.1 to 192.168.1.254

    Example 2: 10.0.0.0/16

    - Network address: 10.0.0.0

    - Broadcast address: 10.0.255.255

    - Usable hosts: 10.0.0.1 to 10.0.255.254

    CIDR and Subnetting

    CIDR enables hierarchical subnetting, allowing networks to be divided into smaller subnets:

    Network: 192.168.0.0/16

    Important: When subnetting, remember that each subnet requires its own network and broadcast addresses, reducing the number of available host addresses by 2 per subnet.

    CIDR in IPv6

    While CIDR was developed for IPv4, the concepts are fundamental to IPv6 addressing:

    • IPv6 uses 128-bit addresses
    • Standard IPv6 subnet size is /64
    • CIDR notation is used similarly (e.g., 2001:db8::/32)
    • Provides even more efficient address aggregation

    Practical Applications

    CIDR is used in:

    • Internet Service Provider (ISP) allocations
    • Cloud computing environments
    • Virtual Private Networks (VPNs)
    • Software Defined Networking (SDN)
    • Network security policies and ACLs
