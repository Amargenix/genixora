    Routing in Computer Networks

    Introduction to Routing

    Routing is the process of selecting a path for traffic in a network or between or across multiple networks. It is performed for many kinds of networks, including circuit-switched networks, such as the public switched telephone network (PSTN), and computer networks, such as the Internet.

    1. Routing Concepts

    Routing vs Forwarding

    • Routing: Determining the path packets take from source to destination
    • Forwarding: Moving packets from a router's input to appropriate output
    • Routing builds forwarding tables

    Routing Metrics

    • Hop count
    • Bandwidth
    • Delay
    • Load
    • Reliability
    • Cost

    Routing Table

    • Destination network
    • Next hop
    • Interface
    • Metric
    • Administrative distance

    2. Routing Algorithms

    Distance Vector

    • Bellman-Ford algorithm
    • Routers share entire routing tables
    • Examples: RIP, IGRP
    • Slow convergence
    • Count-to-infinity problem

    Formula: Dx(y) = minv{c(x,v) + Dv(y)}

    Link State

    • Dijkstra's algorithm
    • Routers share link states
    • Examples: OSPF, IS-IS
    • Fast convergence
    • Requires more memory/CPU

    Formula: Shortest path calculated using link costs

    Path Vector

    • Records complete path information
    • Used for inter-domain routing
    • Example: BGP
    • Policy-based routing
    • Scalable for large networks

    3. Routing Protocols

    Interior Gateway Protocols (IGPs)

    RIP (Routing Information Protocol)

    • Distance vector
    • Hop count metric (max 15)
    • Updates every 30 seconds
    • UDP port 520

    OSPF (Open Shortest Path First)

    • Link state
    • Cost metric (bandwidth-based)
    • Hierarchical areas
    • IP protocol 89

    EIGRP (Enhanced IGRP)

    • Advanced distance vector
    • Composite metric
    • Cisco proprietary
    • IP protocol 88

    Exterior Gateway Protocols (EGPs)

    BGP (Border Gateway Protocol)

    • Path vector
    • Policy-based routing
    • TCP port 179
    • Internet backbone protocol

    BGP Attributes

    • AS Path
    • Next Hop
    • Local Preference
    • MED (Multi-Exit Discriminator)
    • Community

    Route Selection Process

    1. Longest prefix match
    2. Lowest administrative distance
    3. Lowest metric
    4. Other tie-breakers (e.g., oldest route)

    5. Advanced Routing Concepts

    Policy-Based Routing

    • Override normal routing decisions
    • Based on source, protocol, etc.
    • Used for traffic engineering
    • Common in BGP

    Equal-Cost Multi-Path (ECMP)

    • Load balancing across multiple paths
    • Same metric routes
    • Per-packet or per-flow
    • Improves bandwidth utilization

    Software-Defined Networking

    • Decouples control and data planes
    • Centralized routing decisions
    • OpenFlow protocol
    • Dynamic traffic engineering

    6. Routing Security

    Common Threats

    • Route hijacking
    • Route leaks
    • DoS attacks
    • Falsified routing updates

    Protection Mechanisms

    • BGPsec
    • RPKI (Resource Public Key Infrastructure)
    • MD5 authentication
    • Route filtering
