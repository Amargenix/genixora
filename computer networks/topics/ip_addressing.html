    Introduction to IP Addressing

    What is an IP Address?

    An Internet Protocol address (IP address) is a numerical label assigned to each device connected to a computer network that uses the Internet Protocol for communication.

    Key Functions:
    ‚Ä¢ Host or network interface identification
    ‚Ä¢ Location addressing
    ‚Ä¢ Routing of datagrams across network boundaries

    IP Address Characteristics

    • Logical address (not physical/MAC address)
    • Assigned by software (not hardware)
    • Hierarchical structure (network + host portions)
    • Version 4 (32-bit) and Version 6 (128-bit)
    • Can be static or dynamically assigned
    Example IPv4: 192.168.1.10
    Example IPv6: 2001:0db8:85a3:0000:0000:8a2e:0370:7334

    Special IPv4 Addresses

    • 0.0.0.0 - Default route/"this network"
    • 127.0.0.1 - Loopback address
    • 169.254.0.0/16 - APIPA (Automatic Private IP Addressing)
    • 192.168.0.0/16, 172.16.0.0/12, 10.0.0.0/8 - Private networks (RFC 1918)
    • 255.255.255.255 - Limited broadcast

    Subnetting

    Dividing a network into smaller subnetworks using a subnet mask:

    Example: 192.168.1.0/24 (subnet mask 255.255.255.0)

    CIDR Notation

    Classless Inter-Domain Routing (CIDR) represents the subnet mask with a slash followed by the number of network bits:

    Example: 192.168.1.0/26

    ‚Ä¢ Network bits: 26

    IPv6 Addressing

    IPv6 Address Structure

    128-bit address written as eight groups of four hexadecimal digits:

    2001:0db8:85a3:0000:0000:8a2e:0370:7334
    Can be shortened: 2001:db8:85a3::8a2e:370:7334

    IPv6 Address Types

    • Unicast: Identifies a single interface
    • Multicast: Identifies a group of interfaces
    • Anycast: Identifies a set of interfaces (delivered to nearest)

    Special IPv6 Addresses

    • ::1 - Loopback address
    • :: - Unspecified address
    • 2000::/3 - Global unicast
    • fe80::/10 - Link-local addresses
    • fc00::/7 - Unique local addresses (private)
    • ff00::/8 - Multicast addresses

    IP Address Assignment

    Static vs. Dynamic

    Static IP Addressing:

    • Manually configured on each device
    • Never changes unless reconfigured
    • Used for servers, network devices

    Dynamic IP Addressing:

    • Automatically assigned by DHCP server
    • Can change over time (lease duration)
    • Common for client devices

    DHCP Process (DORA)

    1. Discover: Client broadcasts DHCPDISCOVER
    2. Offer: Server responds with DHCPOFFER
    3. Request: Client requests offered address (DHCPREQUEST)
    4. Acknowledge: Server confirms with DHCPACK

    APIPA

    Automatic Private IP Addressing (169.254.0.0/16) is assigned when DHCP fails, allowing local network communication.

    NAT and Public vs. Private IPs

    Private IP Addresses

    Defined in RFC 1918 for use in private networks:

    • 10.0.0.0/8 (16,777,216 hosts)
    • 172.16.0.0/12 (1,048,576 hosts)
    • 192.168.0.0/16 (65,536 hosts)

    Public IP Addresses

    Globally routable addresses assigned by IANA and regional registries (ARIN, RIPE, etc.). Required for Internet communication.

    Network Address Translation (NAT)

    Allows multiple devices on a private network to share a single public IP address:

    • Static NAT: One-to-one mapping
    • Dynamic NAT: Pool of public addresses
    • PAT (NAT Overload): Many-to-one using ports
