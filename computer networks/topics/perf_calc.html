    Prefix Calculation in Computer Networks

    Introduction to Prefix Calculation

    Prefix calculation is fundamental to IP addressing and subnetting in computer networks. It involves determining network portions of IP addresses, calculating subnet sizes, and optimizing address allocation using CIDR (Classless Inter-Domain Routing) notation.

    1. CIDR Notation Basics

    Understanding CIDR Format

    CIDR notation combines an IP address with a suffix indicating the network prefix length:

    IPv4 Example

    192.168.1.0/24

    Where /24 means first 24 bits are network prefix

    IPv6 Example

    2001:db8::/32

    Where /32 means first 32 bits are network prefix

    Key Concept: The prefix length indicates how many bits of the address define the network portion, with the remaining bits used for host addressing.

    Hosts per Prefix

    Calculate maximum hosts for a given prefix length:

    IPv4: Hosts = 2(32 - prefix) - 2
    IPv6: Hosts = 2(128 - prefix)
    Example: /28 prefix in IPv4:
    Hosts = 2(32-28) - 2 = 16 - 2 = 14

    Prefix to Subnet Mask

    Convert prefix length to dotted-decimal subnet mask:

    Mask = 232 - 2(32 - prefix)
    Example: /26 prefix:
    Mask = 232 - 26 = 255.255.255.192

    4. VLSM Calculations

    Variable Length Subnet Masking

    VLSM allows different subnet masks in the same network for efficient address space utilization.

    VLSM Calculation Steps

    1. Sort subnet requirements from largest to smallest
    2. Allocate largest subnets first
    3. Continue with progressively smaller subnets
    4. Track allocated address ranges

    5. Route Aggregation (Supernetting)

    Calculating Aggregate Routes

    Route aggregation combines multiple contiguous networks into a single summarized route.

    Aggregation Steps

    1. List networks in binary
    2. Find longest common prefix
    3. Count matching bits for new prefix
    4. Verify all networks fall within aggregate

    Aggregation Example

    Aggregate these networks:
    192.168.0.0/24
    192.168.1.0/24
    192.168.2.0/24
    192.168.3.0/24

    Solution:
    Binary: 11000000.10101000.00000000.00000000
    11000000.10101000.00000001.00000000
    11000000.10101000.00000010.00000000
    11000000.10101000.00000011.00000000

    Common prefix: first 22 bits
    Aggregate: 192.168.0.0/22

    6. IPv6 Prefix Calculations

    IPv6 Addressing Concepts

    IPv6 uses 128-bit addresses with different prefix calculation considerations.

    IPv6 Prefix Allocation

    • Global unicast: Typically /48 per site
    • Subnets: Typically /64
    • Interface ID: Last 64 bits
    • No broadcast addresses

    IPv6 Example

    Given: 2001:db8:acad::/48

    Subnet 1: 2001:db8:acad:1::/64
    Subnet 2: 2001:db8:acad:2::/64
    Subnet 3: 2001:db8:acad:3::/64

    IPv6 Prefix Calculation

    Subnets = 2(subnet_bits)
    Where subnet_bits = new_prefix - original_prefix
    Example: From 2001:db8::/32 to /48 subnets:
    Subnet bits = 48 - 32 = 16
    Possible subnets = 216 = 65,536
