Computer Networks

    FTP and SMTP Protocols

    FTP (File Transfer Protocol)

    FTP is a standard network protocol used for the transfer of computer files between a client and server on a computer network.

    Key Characteristics

    • Port Numbers: 20 (data), 21 (control)
    • Protocol Type: TCP
    • Transfer Modes: Active and Passive
    • Authentication: Username/password (anonymous available)
    • Commands: ASCII/text or binary

    FTP Communication Model

    Client (Port X) ↔ Server (Port 21 - Control)
    Client (Port X+1) ↔ Server (Port 20 - Data)

    FTP Commands

    USER - Username
    PASS - Password
    LIST - List files
    RETR - Retrieve file
    STOR - Store file
    DELE - Delete file
    CWD - Change directory
    QUIT - Logout

    FTP Variants

    FTPS (FTP Secure)
    SFTP (SSH FTP)
    TFTP (Trivial FTP)
    FTP over HTTP

    Security Considerations

    Standard FTP transmits data and credentials in clear text. For secure transfers, consider:

    • FTPS (FTP over SSL/TLS)
    • SFTP (SSH File Transfer Protocol)
    • VPN tunnel for FTP connections

    SMTP (Simple Mail Transfer Protocol)

    SMTP is the standard protocol for sending emails across IP networks. It handles message routing between mail servers.

    Key Characteristics

    • Port Numbers: 25 (standard), 587 (submission), 465 (SMTPS)
    • Protocol Type: TCP
    • Communication Model: Store-and-forward
    • Commands: Text-based
    • Security: Originally plaintext, now often with STARTTLS

    SMTP Communication Flow

    1. HELO/EHLO - Identify client
    2. MAIL FROM - Sender address
    3. RCPT TO - Recipient address
    4. DATA - Message content
    5. QUIT - End session

    SMTP Response Codes

    2xx - Success
    220 - Service ready
    250 - Request completed
    4xx - Temporary error
    5xx - Permanent error
    554 - Transaction failed

    SMTP Extensions

    ESMTP
    STARTTLS
    SMTP-AUTH
    DKIM
    SPF
    DMARC

    Email Delivery Process

    1. Sender's email client → Sender's SMTP server
    2. Sender's SMTP server → Recipient's SMTP server
    3. Recipient's SMTP server → Recipient's mailbox
    4. Recipient retrieves email via POP3/IMAP

    Security Considerations

    SMTP originally had no security features. Modern implementations add:

    • STARTTLS for encryption
    • SMTP-AUTH for authentication
    • SPF/DKIM/DMARC for sender verification
    • Rate limiting to prevent spam

    Comparison: FTP vs SMTP

    Feature FTP SMTP
    Primary Purpose File transfer Email transmission
    Port Numbers 20 (data), 21 (control) 25, 587, 465
    Connection Type Dual connection (control + data) Single connection
    Transfer Mode Binary or ASCII Text-based commands with MIME encoding
    Authentication Username/password (or anonymous) Originally none, now SMTP-AUTH
    Security FTPS, SFTP STARTTLS, SMTPS

    Practical Applications

    FTP Use Cases

    • Website maintenance and updates
    • Large file transfers between organizations
    • Software distribution
    • Backup and archival systems
    • Bulk data transfers

    SMTP Use Cases

    • Email delivery between servers
    • Transactional email systems
    • Mailing lists
    • Email marketing platforms
    • System notification services

    Protocol Commands Examples

    FTP Session Example

    $ ftp example.com
    Connected to example.com.
    220 FTP Server ready.
    Name: username
    331 Password required.
    Password: ********
    230 User logged in.
    ftp> ls
    200 PORT command successful.
    150 Opening ASCII mode data connection.
    -rw-r--r-- 1 user group 1234 Jan 1 file.txt
    226 Transfer complete.
    ftp> get file.txt
    200 PORT command successful.
    150 Opening BINARY mode data connection.
    226 Transfer complete.
    ftp> quit
    221 Goodbye.
