    Routing Tables

    Introduction to Routing Tables

    A routing table is a data table stored in a router or network host that lists the routes to particular network destinations. It contains information about the topology of the network immediately around it and determines where packets will be forwarded based on their destination IP addresses.

    Routing tables are fundamental to the operation of IP networks, enabling routers to make decisions about the best path for forwarding packets toward their destinations.

    Key Characteristics

    • Stored in router's memory (RAM)
    • Dynamic (updated by routing protocols) or static (manually configured)
    • Contains multiple routes for redundancy
    • Used for both IPv4 and IPv6 (separate tables)
    • Optimized for fast lookup

    Routing Table Components

    Destination Network

    The IP address of the destination network or host.

    • Can be a specific host (/32) or network (with subnet mask)
    • 0.0.0.0/0 is the default route
    • 127.0.0.0/8 is for loopback addresses

    Subnet Mask

    Determines which portion of the IP address is the network portion.

    • Used with the destination to identify the network
    • Helps determine route specificity
    • Longest prefix match determines route selection

    Next Hop

    The IP address of the next router to which packets should be forwarded.

    • Can be an interface for directly connected networks
    • "On-link" means directly reachable
    • May be the local router's own interface

    Interface

    The local network interface used to reach the next hop.

    • Physical or logical interface (Ethernet, VLAN, etc.)
    • Must be operational for the route to be valid
    • Used for directly connected routes

    Metric

    A value used to determine the best route when multiple paths exist.

    • Lower values indicate preferred routes
    • Common metrics: hop count, bandwidth, delay, reliability
    • Administrative Distance (AD) determines protocol preference

    Route Source

    How the route was learned (protocol or manual configuration).

    • Directly connected (C)
    • Static (S)
    • Dynamic (OSPF, EIGRP, BGP, etc.)
    • Default (gateway of last resort)

    Types of Routing Table Entries

    Directly Connected Routes

    Routes to networks directly attached to the router's interfaces.

    • Automatically added when interface is configured
    • Removed when interface is down
    • Administrative Distance: 0
    • Most preferred route type

    Static Routes

    Manually configured routes entered by network administrators.

    • No overhead (unlike dynamic routing)
    • Administrative Distance: 1
    • Require manual updates for topology changes
    • Useful for small networks or specific routes

    Dynamic Routes

    Routes learned through routing protocols (OSPF, EIGRP, BGP, etc.).

    • Automatically adapt to topology changes
    • Different AD values per protocol
    • Require CPU and bandwidth resources
    • Essential for large, complex networks

    Default Route

    A catch-all route used when no specific route exists.

    • Destination: 0.0.0.0/0
    • Points to "gateway of last resort"
    • Common in stub networks
    • Can be static or learned via protocols

    Host Routes

    Routes to a specific host (32-bit mask for IPv4, 128-bit for IPv6).

    • Most specific route possible
    • Used for special cases or diagnostics
    • Can be created manually or automatically
    • Take precedence over network routes

    Floating Static Routes

    Backup routes with higher AD than primary routes.

    • Become active when primary route fails
    • AD must be higher than dynamic routes
    • Used for redundancy
    • Often point to backup links

    Route Selection Process

    1. Longest Prefix Match

    The router selects the route with the most specific (longest) subnet mask.

    • Host route (/32) is most specific
    • Default route (/0) is least specific
    • Example: /24 is preferred over /16 for same destination

    2. Administrative Distance

    If multiple routes have same prefix length, the one with lowest AD is chosen.

    • Directly connected: 0
    • Static: 1
    • EIGRP: 90
    • OSPF: 110
    • RIP: 120

    3. Metric

    If AD is same, the route with best metric is selected.

    • OSPF: Cost (based on bandwidth)
    • EIGRP: Composite metric
    • RIP: Hop count
    • BGP: Path attributes

    Route Selection Example

    Destination AD Metric Selected? Reason
    10.1.1.0/24 0 - Yes Directly connected (lowest AD)
    10.1.1.1/32 1 - No Longer prefix but higher AD
    10.1.0.0/16 110 20 No Less specific prefix
    10.1.1.0/24 110 10 No Same prefix but higher AD

    Routing Table Management

    Viewing Routing Tables

    • Cisco IOS: show ip route
    • Linux: ip route or route -n
    • Windows: route print
    • Junos: show route

    Adding Static Routes

    • Cisco: ip route network mask next-hop
    • Linux: ip route add network/mask via next-hop
    • Windows: route add network mask next-hop
    • Persistent routes: Need special configuration

    Route Redistribution

    • Sharing routes between different routing protocols
    • Requires careful metric translation
    • Can cause routing loops if not configured properly
    • Filtering is often necessary

    Route Filtering

    • Access Control Lists (ACLs)
    • Prefix lists
    • Route maps
    • Distribute lists

    Advanced Routing Table Concepts

    Equal-Cost Multi-Path (ECMP)

    Using multiple paths with same metric for load balancing

    Route Recursion

    Process of resolving next-hop IP addresses to exit interfaces

    Virtual Routing and Forwarding (VRF)

    Multiple routing tables for network segmentation

    Policy-Based Routing

    Making routing decisions based on policies beyond destination IP
