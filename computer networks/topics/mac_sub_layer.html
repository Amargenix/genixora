    MAC Sublayer in Computer Networks

    Introduction to MAC Sublayer

    What is the MAC Sublayer?

    The MAC (Media Access Control) sublayer is the lower portion of the Data Link Layer (Layer 2) in the OSI model. It provides addressing and channel access control mechanisms for network nodes to communicate.

    Key Responsibilities:
    • Frame addressing and recognition
    • Media access control
    • Error detection (but not correction)
    • Logical topology implementation

    MAC Addressing

    MAC Address Structure

    A MAC address is a 48-bit (6-byte) identifier typically represented in hexadecimal notation:

    00:1A:2B:3C:4D:5E or 00-1A-2B-3C-4D-5E
    Part Bits Description
    OUI 24 Organizationally Unique Identifier (assigned by IEEE)
    NIC 24 Network Interface Controller specific

    MAC Address Types

    • Unicast: Directed to a specific device
    • Multicast: Directed to a group of devices
    • Broadcast: FF:FF:FF:FF:FF:FF (all devices on segment)

    MAC vs IP Addressing

    Feature MAC Address IP Address
    Layer Data Link (L2) Network (L3)
    Assigned by Manufacturer Network admin/DHCP
    Format 48-bit hexadecimal 32/128-bit decimal/hex
    Scope Local network Global/local
    Changeable Hardware-based (mostly fixed) Software-configurable

    MAC Address Lookup

    The first 3 bytes (OUI) can identify the manufacturer:

    Example: 00:1A:2B ‚Üí Google, Inc.

    Tools like Wireshark or online OUI lookup can identify vendors.

    Media Access Control Methods

    Controlled Access

    Devices take turns accessing the medium:

    Polling

    A primary device controls access by polling secondary devices.

    Token Passing

    Devices can transmit only when holding a token (used in Token Ring and FDDI).

    Advantages: No collisions, deterministic
    Disadvantages: Token overhead, single point of failure

    TDMA/FDMA/CDMA

    Time/Frequency/Code Division Multiple Access techniques used in wireless networks.

    Random Access

    Devices compete for medium access:

    ALOHA

    Original random access protocol (pure and slotted versions).

    CSMA

    Carrier Sense Multiple Access with variants:

    • CSMA/CD: Collision Detection (Ethernet)
    • CSMA/CA: Collision Avoidance (Wi-Fi)
    CSMA/CD Process (Ethernet):
    1. Listen before transmitting
    2. If idle, transmit
    3. If collision detected, send jam signal
    4. Wait random backoff time, then retry

    Ethernet MAC Sublayer

    Ethernet Frame Structure

    Field Size Description
    Preamble 7 bytes Synchronization
    SFD 1 byte Start Frame Delimiter
    Destination MAC 6 bytes Receiver address
    Source MAC 6 bytes Sender address
    Type/Length 2 bytes EtherType or frame length
    Data 46-1500 bytes Payload
    FCS 4 bytes Frame Check Sequence (CRC)

    Ethernet MAC Operation

    Key aspects of Ethernet MAC sublayer:

    MAC Address Learning

    Switches build MAC address tables by examining source addresses of incoming frames.

    Frame Forwarding

    Switches use destination MAC addresses to forward frames to appropriate ports.

    Frame Filtering

    Frames are filtered if source and destination are on the same switch port.

    Broadcast Handling

    Broadcast frames (FF:FF:FF:FF:FF:FF) are flooded to all ports except the incoming port.

    Minimum Frame Size: 64 bytes (to ensure collision detection)
    Maximum Frame Size: 1518 bytes (1522 with VLAN tagging)

    Wireless MAC Sublayer

    IEEE 802.11 MAC

    Wireless networks use CSMA/CA (Collision Avoidance) instead of CSMA/CD:

    • Hidden Node Problem: Solved with RTS/CTS (Request to Send/Clear to Send)
    • DCF: Distributed Coordination Function (default mode)
    • PCF: Point Coordination Function (optional polling mode)

    Wi-Fi Frame Structure

    Contains more fields than Ethernet to handle wireless complexities:

    • Frame Control
    • Duration/ID
    • Address fields (up to 4 addresses)
    • Sequence Control
    • QoS Control
    • HT Control

    CSMA/CA Process

    1. Listen to determine if channel is idle (physical + virtual carrier sense)
    2. If busy, wait until channel becomes idle plus DIFS (DCF Interframe Space)
    3. Choose random backoff time from contention window
    4. Count down backoff timer while channel remains idle
    5. If timer reaches zero, transmit frame
    6. Wait for ACK (if not received, double contention window and retry)
    Virtual Carrier Sense: Uses NAV (Network Allocation Vector) in frame headers to reserve medium.

    Wi-Fi MAC Address Types

    Wireless frames can contain up to 4 MAC addresses:

    1. Receiver Address (RA)
    2. Transmitter Address (TA)
    3. Source Address (SA)
    4. Destination Address (DA)
