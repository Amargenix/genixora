    1. Software-Defined Networking (SDN)

    SDN Architecture

                        +-----------------------+
                    |   Application Layer   |
                    | (Network Applications)|
                    +----------+------------+
                               |
                               | Northbound API
                               |
                    +----------+------------+
                    |   Control Layer       |
                    | (SDN Controller)      |
                    +----------+------------+
                               |
                               | Southbound API (OpenFlow)
                               |
                    +----------+------------+
                    |   Infrastructure      |
                    | (Data Plane Switches) |
                    +-----------------------+

    SDN Key Components

    Data Plane

    • Forwarding devices (switches, routers)
    • Flow tables with match-action rules
    • Hardware acceleration (TCAM)

    Control Plane

    • Centralized SDN controller
    • Network operating system
    • Topology management

    Northbound API

    • RESTful APIs
    • Application integration
    • Intent-based networking

    SDN Protocols and Implementations

    Protocol Standard Key Features
    OpenFlow ONF Original SDN protocol, flow tables
    OVSDB RFC 7047 Open vSwitch management
    P4 P4.org Programming protocol-independent processors
    NETCONF/YANG RFC 6241 Network configuration protocol

    SDN Use Cases

    • Campus networks: Dynamic policy enforcement
    • Data centers: Network virtualization
    • WAN: Traffic engineering
    • 5G: Network slicing

    NFV vs Traditional Networking

    Aspect Traditional NFV
    Deployment Hardware appliances Software on COTS servers
    Scaling Physical capacity limits Elastic scaling
    Upgrades Hardware replacement Software updates
    Cost High CapEx Lower TCO

    NFV Implementation Challenges

    Performance

    • Packet processing overhead
    • NUMA architecture awareness
    • SR-IOV vs virtio tradeoffs

    Orchestration

    • Service chaining complexity
    • VNF lifecycle management
    • Multi-vendor interoperability

    Security

    • Hypervisor vulnerabilities
    • East-west traffic protection
    • VNF isolation

    NFV Acceleration Techniques

    • DPDK: Userspace packet processing
    • SR-IOV: Direct hardware access
    • SmartNICs: Offload processing
    • eBPF: Kernel-level filtering

    3. Intent-Based Networking (IBN)

    IBN System Workflow

                        +-------------------+       +-------------------+       +-------------------+
                    |                   |       |                   |       |                   |
                    |  Translate        |------>|  Automate         |------>|  Assure           |
                    |  Business Intent  |       |  Configuration    |       |  Compliance       |
                    |                   |       |                   |       |                   |
                    +-------------------+       +-------------------+       +-------------------+
                            ^                                                        |
                            |                                                        |
                            +--------------------------------------------------------+
                                            Continuous Validation and Adjustment

    IBN Key Components

    Component Function Technologies
    Translation Convert intent to policies NLP, Machine Learning
    Activation Implement network changes APIs, NETCONF, Ansible
    Assurance Verify intent compliance Telemetry, AI Analytics

    IBN Implementation Considerations

    • Data collection: Streaming telemetry vs polling
    • Modeling: YANG data models for network state
    • Closed-loop: Feedback mechanisms for self-healing
    • Security: Zero-trust principles integration

    IBN Use Cases

    Automated Provisioning

    "Connect all IoT devices in Building A with 10Mbps guaranteed bandwidth"

    Security Enforcement

    "Block all unauthorized devices from accessing financial systems"

    Performance Optimization

    "Ensure video conferencing gets priority during business hours"

    Technique Layer Impact
    TCP BBR Transport Higher throughput, lower latency

    • Diagnostics: Wireshark, TCPdump
