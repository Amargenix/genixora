    Distributed Denial of Service (DDoS) Attacks

    Introduction to DDoS Attacks

    A Distributed Denial of Service (DDoS) attack is a malicious attempt to disrupt normal traffic of a targeted server, service, or network by overwhelming the target or its surrounding infrastructure with a flood of Internet traffic from multiple sources.

    Key Characteristics

    • Originates from multiple compromised systems
    • Difficult to block due to distributed nature
    • Can generate traffic volumes exceeding 1 Tbps
    • Often uses botnets (networks of infected devices)

    Impact of DDoS Attacks

    • Service downtime and unavailability
    • Financial losses from disrupted operations
    • Reputation damage and loss of customer trust
    • Can serve as smokescreen for other attacks

    Motivations

    • Extortion (ransom demands)
    • Competitive advantage
    • Hacktivism (political/social motives)
    • Cyber warfare (nation-state attacks)

    Types of DDoS Attacks

    1. Volume-Based Attacks

    Saturate the bandwidth of the target site by flooding it with massive amounts of traffic.

    1 UDP Flood

    Floods random ports on the target with UDP packets, causing the host to repeatedly check for applications listening at those ports.

    Amplification factor: 10x-100x

    2 ICMP (Ping) Flood

    Overwhelms the target with ICMP Echo Request (ping) packets, consuming both incoming and outgoing bandwidth.

    3 DNS Amplification

    Exploits open DNS resolvers to turn small queries into much larger responses directed at the target.

    Example: Using DNS ANY queries for amplification

    2. Protocol Attacks

    Exploit weaknesses in network protocols to consume server resources or intermediate communication equipment.

    SYN Flood

    Exploits TCP handshake by sending SYN requests but never completing the handshake, exhausting connection tables.

    Attackers spoof source IPs to prevent SYN-ACK responses

    Ping of Death

    Sends malformed or oversized packets using ICMP, crashing vulnerable systems.

    Slowloris

    Keeps many connections to the target open by sending partial HTTP requests, exhausting connection pools.

    3. Application Layer Attacks

    Target specific applications with seemingly legitimate requests that consume excessive resources.

    HTTP Flood

    Sends seemingly legitimate HTTP GET or POST requests to overwhelm a web server.

    Variants: GET floods, POST floods

    DNS Query Flood

    Targets DNS servers with rapid, repeated queries for random subdomains.

    SSL/TLS Exhaustion

    Exploits the computational cost of SSL/TLS handshakes to consume server resources.

    DDoS Attack Anatomy

    Attack Lifecycle

    1. 1 Botnet Creation: Infect devices with malware to create a network of bots
    2. 2 Command & Control: Attacker sends instructions to botnet through C2 servers
    3. 3 Attack Launch: Bots simultaneously send malicious traffic to target
    4. 4 Target Impact: Service becomes unavailable due to resource exhaustion
    5. 5 Attack Sustained: Continues until mitigated or attacker stops

    Botnet Components

    Botmaster: Attacker controlling the botnet
    C2 Servers: Command and control infrastructure
    Zombies/Bots: Compromised devices (IoT, servers, PCs)
    Reflectors: Servers used in amplification attacks

    Attack Vectors

    Direct Attack: Bots send traffic directly to target

    Reflective Attack: Spoofed requests to third-party servers that respond to target

    DDoS Protection & Mitigation

    Preventive Measures

    • Network Architecture: Redundancy and load balancing
    • Rate Limiting: Restrict requests per IP/second
    • Web Application Firewall: Filter malicious traffic
    • Blackhole Routing: Drop traffic at router level
    • Anycast Distribution: Spread traffic across servers

    Detection Techniques

    • Traffic Analysis: Baseline normal traffic patterns
    • Anomaly Detection: Identify unusual traffic spikes
    • Behavioral Analysis: Detect bot-like behavior
    • Geolocation: Flag traffic from unusual locations
    • IP Reputation: Block known malicious IPs

    Mitigation Strategies

    • Cloud-based Protection: Services like Cloudflare, Akamai
    • Traffic Scrubbing: Filter malicious packets
    • Sinkholing: Redirect attack traffic
    • ISP Coordination: Block at network edge
    • Incident Response Plan: Prepared procedures

    Best Practices for Organizations

    Capacity Planning

    Ensure sufficient bandwidth headroom

    DDoS Testing

    Regular simulated attack exercises

    Vendor Selection

    Choose providers with DDoS protection

    Monitoring

    24/7 network traffic monitoring

    Emerging DDoS Trends

    IoT Botnets

    Exploiting insecure IoT devices (cameras, routers) to create massive botnets like Mirai and its variants.

    Multi-Vector Attacks

    Combining multiple attack types simultaneously to bypass defenses (e.g., volumetric + application layer).

    AI-Powered Attacks

    Using machine learning to adapt attack patterns in real-time to evade detection systems.

    Ransom DDoS (RDoS)

    Extortion threats combined with demonstration attacks to pressure victims into paying.

    5G Vulnerabilities

    Potential for larger-scale attacks as more high-bandwidth devices connect to 5G networks.
