    Firewalls in Computer Networks

    What is a Firewall?

    A firewall is a network security device or software that monitors and controls incoming and outgoing network traffic based on predetermined security rules. It establishes a barrier between trusted internal networks and untrusted external networks (such as the Internet).

    Types of Firewalls

    1. Packet-Filtering Firewalls

    • Operate at network layer (Layer 3)
    • Examine packets in isolation
    • Filter based on IP addresses, ports, protocols
    • Fast but limited security

    2. Stateful Inspection Firewalls

    • Track connection state (Layer 4)
    • Remember previous packets in conversation
    • More secure than packet filtering
    • Common in modern networks

    3. Proxy Firewalls

    • Operate at application layer (Layer 7)
    • Terminate and re-establish connections
    • Deep packet inspection
    • High security but slower

    4. Next-Generation Firewalls (NGFW)

    • Combine traditional firewall with advanced features
    • Application awareness
    • Integrated intrusion prevention
    • User identity integration

    5. Web Application Firewalls (WAF)

    • Specialized for web applications
    • Protect against OWASP Top 10 vulnerabilities
    • Understand HTTP/HTTPS traffic
    • Often cloud-based

    6. Unified Threat Management (UTM)

    • All-in-one security appliances
    • Combine firewall with antivirus, spam filtering, etc.
    • Good for small to medium businesses
    • Simplified management

    Firewall Deployment Architectures

    1. Network Perimeter Firewall

    Placed at the boundary between internal network and Internet. Protects entire organization from external threats.

    Internet → Firewall → Internal Network

    2. DMZ Configuration

    Uses two firewalls to create a demilitarized zone for public-facing servers.

    Internet → Front Firewall → DMZ → Back Firewall → Internal Network

    3. Internal Segmentation

    Firewalls between different internal network segments to limit lateral movement.

    Finance Dept → Firewall → Engineering Dept

    4. Cloud Firewalls

    Virtual firewalls protecting cloud infrastructure and virtual networks.

    Public Cloud → Virtual Firewall → VPC/Cloud Resources

    Firewall Features and Capabilities

    Core Features

    • Packet filtering
    • Network Address Translation (NAT)
    • Stateful inspection
    • VPN support
    • Logging and monitoring
    • DoS/DDoS protection

    Advanced Features (NGFW)

    • Application awareness and control
    • User identity integration
    • Integrated intrusion prevention (IPS)
    • SSL/TLS inspection
    • Threat intelligence feeds
    • Sandboxing for unknown threats

    Firewall Implementation Considerations

    Selection Criteria

    • Network size and throughput requirements
    • Required security features
    • Management interface preferences
    • Integration with existing systems
    • Vendor support and reputation
    • Total cost of ownership

    Best Practices

    • Default deny policy
    • Regular rule base reviews
    • Document all rule changes
    • Separate development and production environments
    • Implement high availability where needed
    • Regular firmware/software updates

    Firewall Limitations

    While essential, firewalls have limitations that organizations should understand:

    1
    Cannot prevent insider threats
    2
    Limited against encrypted threats
    3
    Cannot stop social engineering attacks
    4
    May create performance bottlenecks
    5
    Require skilled administration
    6
    Cannot protect against zero-day exploits
