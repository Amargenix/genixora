    Domain Name System (DNS)

    What is DNS?

    DNS (Domain Name System) is a hierarchical and decentralized naming system used to translate human-readable domain names (like www.example.com) into machine-readable IP addresses (like 192.0.2.1).

    Key characteristics:

    • Acts as the internet's phonebook
    • Uses a distributed database architecture
    • Operates on UDP port 53 (TCP for larger responses)
    • Essential for internet functionality

    How DNS Works

    The DNS resolution process involves several steps:

    1. User enters a URL in the browser
    2. Browser checks its cache
    3. OS checks its hosts file and cache
    4. Query sent to recursive DNS resolver
    5. Resolver queries root nameservers
    6. Root directs to TLD nameservers
    7. TLD directs to authoritative nameservers
    8. Authoritative server provides IP address
    9. Resolver returns IP to browser

    2. TLD: ".com"

    3. Authoritative: "example.com"

    DNS Components

    DNS Resolver

    Also called recursive resolver. Typically operated by ISPs or public services like Google DNS (8.8.8.8) or Cloudflare (1.1.1.1).

    Root Nameservers

    13 logical root servers (with hundreds of physical instances worldwide) that know where to direct queries for TLDs.

    TLD Nameservers

    Servers for top-level domains like .com, .org, .net, and country codes like .uk, .jp.

    Authoritative Nameservers

    Servers that contain actual DNS records for domains. Managed by domain owners or their DNS providers.

    Record Type Purpose Example

    AAAA IPv6 address record example.com AAAA 2606:2800:220:1:248:1893:25c8:1946

    DNS Caching

    DNS uses caching extensively to improve performance and reduce load:

    • Browser Cache: Modern browsers cache DNS records
    • OS Cache: Operating systems maintain DNS caches
    • Resolver Cache: Recursive resolvers cache responses based on TTL (Time To Live)

    TTL (Time To Live): Specified in seconds, determines how long a record can be cached. Example: "example.com A 3600" means the record can be cached for 1 hour (3600 seconds).

    DNS Security

    DNS Spoofing/Cache Poisoning

    Attackers corrupt DNS cache with false information, redirecting users to malicious sites.

    DNSSEC (DNS Security Extensions)

    Adds cryptographic signatures to DNS records to verify their authenticity.

    DNS over HTTPS (DoH) / DNS over TLS (DoT)

    Encrypts DNS queries to prevent eavesdropping and manipulation.

    DNS Tools and Commands

    Command Line Tools

    • nslookup - Query DNS servers
    • dig - Advanced DNS lookup tool
    • host - Simple DNS lookup utility

    Online Tools

    • DNS lookup tools (e.g., MXToolbox)
    • DNS propagation checkers
    • DNSSEC validators

    Example Commands

    $ nslookup example.com
