    Variable Length Subnet Mask (VLSM)

    VLSM is a technique that allows network administrators to allocate IP addresses more efficiently by using subnet masks of different lengths for different subnets within the same network.

    Basic Concepts

    Traditional Subnetting vs VLSM

    • Fixed Length Subnet Mask (FLSM): All subnets have the same size, leading to IP address wastage
    • Variable Length Subnet Mask (VLSM): Subnets can have different sizes based on requirements, minimizing IP address wastage

    How VLSM Works

    1. Determine requirements: Identify how many hosts are needed in each subnet
    2. Sort subnets: Arrange subnets in descending order based on host requirements
    3. Allocate largest first: Assign the largest subnet from the available address space
    4. Continue allocation: Move to the next largest subnet and allocate from remaining space
    5. Repeat: Continue until all subnets are allocated

    VLSM Calculation Example

    Given network: 192.168.1.0/24

    Subnet requirements:

    • Subnet A: 60 hosts
    • Subnet B: 30 hosts
    • Subnet C: 12 hosts
    • Subnet D: 5 hosts
    Subnet Hosts Needed Subnet Mask Address Range Broadcast Address
    A 60 /26 (255.255.255.192) 192.168.1.0 - 192.168.1.63 192.168.1.63
    B 30 /27 (255.255.255.224) 192.168.1.64 - 192.168.1.95 192.168.1.95
    C 12 /28 (255.255.255.240) 192.168.1.96 - 192.168.1.111 192.168.1.111
    D 5 /29 (255.255.255.248) 192.168.1.112 - 192.168.1.119 192.168.1.119

    Advantages of VLSM

    • Efficient use of IP address space
    • Reduces IP address wastage
    • Allows hierarchical addressing
    • Better network organization
    • Supports route summarization

    Disadvantages of VLSM

    • More complex to implement than FLSM
    • Requires careful planning
    • Troubleshooting can be more difficult
    • Not all legacy routing protocols support VLSM

    Practical Applications

    • Enterprise networks with varying subnet size requirements
    • Internet Service Providers (ISPs) allocating addresses to customers
    • Data center networks with different sized segments
    • Any environment where IP address conservation is important

    VLSM and CIDR

    VLSM is closely related to Classless Inter-Domain Routing (CIDR):

    • Both allow for more flexible address allocation
    • CIDR is used for routing between networks (supernetting)
    • VLSM is used within a single network (subnetting)
    • Modern routing protocols (OSPF, EIGRP, BGPv4) support both

    Key Takeaways

    • VLSM allows different subnet masks in the same network
    • Allocate largest subnets first for efficient space utilization
    • Essential for modern network design and IP conservation
    • Requires careful planning and documentation
