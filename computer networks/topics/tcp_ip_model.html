    TCP/IP Model in Computer Networks

    Introduction to TCP/IP Model

    The TCP/IP model is a concise version of the OSI model. It contains four layers, unlike the seven layers in the OSI model. The layers are:

    1. Application Layer
    2. Transport Layer
    3. Internet Layer
    4. Network Access Layer

    The TCP/IP model was developed by the U.S. Department of Defense (DoD) in the 1970s and forms the basis of the modern internet.

    TCP/IP Layers Explained

    1. Application Layer

    This layer performs the functions of the top three layers of the OSI model: Application, Presentation, and Session layers.

    Protocols: HTTP, FTP, SMTP, DNS, Telnet, SSH

    Functions:

    • Provides user interfaces and support for services
    • Handles network transparency, resource allocation, etc.
    • Enables communication between applications

    2. Transport Layer

    This layer is responsible for end-to-end communication and error-free delivery of data.

    Protocols: TCP, UDP

    Functions:

    • Segmentation and reassembly of data
    • Flow control and error control
    • Connection-oriented (TCP) and connectionless (UDP) communication
    • Port addressing (using port numbers)

    TCP vs UDP:

    TCP (Transmission Control Protocol): Reliable, connection-oriented, slower, provides error checking and recovery

    UDP (User Datagram Protocol): Unreliable, connectionless, faster, no error checking or recovery

    3. Internet Layer

    Also known as the Network Layer, it handles the movement of packets around the network.

    Protocols: IP, ICMP, ARP, RARP, IGMP

    Functions:

    • Logical addressing (IP addresses)
    • Routing of packets across networks
    • Fragmentation and reassembly of packets
    • Handles congestion control

    IP (Internet Protocol): Responsible for delivering packets from the source host to the destination host based on IP addresses.

    4. Network Access Layer

    Also called the Network Interface Layer or Link Layer, it combines the Physical and Data Link layers of the OSI model.

    Protocols: Ethernet, Token Ring, PPP, HDLC, Frame Relay

    Functions:

    • Physical addressing (MAC addresses)
    • Encoding and signaling
    • Transmission of raw bits over the communication channel
    • Error detection in transmitted data

    TCP/IP vs OSI Model

    OSI Model TCP/IP Model
    7 Layers 4 Layers
    Separate Presentation and Session layers Combined in Application layer
    Separate Data Link and Physical layers Combined in Network Access layer
    Protocol independent standard Protocol dependent
    Developed by ISO Developed by DoD

    TCP/IP Protocol Suite

    Application Layer Protocols

    • HTTP: Hypertext Transfer Protocol (web)
    • FTP: File Transfer Protocol
    • SMTP: Simple Mail Transfer Protocol
    • DNS: Domain Name System
    • SNMP: Simple Network Management Protocol

    Transport Layer Protocols

    • TCP: Connection-oriented reliable protocol
    • UDP: Connectionless unreliable protocol
    • SCTP: Stream Control Transmission Protocol

    Internet Layer Protocols

    • IP: Internet Protocol (IPv4, IPv6)
    • ICMP: Internet Control Message Protocol
    • ARP: Address Resolution Protocol
    • RARP: Reverse ARP

    Network Access Protocols

    • Ethernet: Most common LAN technology
    • PPP: Point-to-Point Protocol
    • HDLC: High-Level Data Link Control

    TCP/IP Encapsulation Process

    Data moves through the TCP/IP layers as follows:

    1. Application Layer: Creates the message or data to be sent
    2. Transport Layer: Adds TCP/UDP header (becomes segment/datagram)
    3. Internet Layer: Adds IP header (becomes packet)
    4. Network Access Layer: Adds frame header and trailer (becomes frame)
    Application Data
    TCP/UDP Header | Application Data
    IP Header | TCP/UDP Header | Application Data
    Frame Header | IP Header | TCP/UDP Header | Application Data | Frame Trailer

    Key Features of TCP/IP

    • Packet Switching: Data is broken into packets that travel independently
    • End-to-End Connectivity: Provides connectivity between any two nodes
    • Scalability: Can scale to large networks like the internet
    • Dynamic Routing: Adapts to changes in network topology
    • Robustness: Can handle network failures gracefully
    • Interoperability: Works across different hardware and software
