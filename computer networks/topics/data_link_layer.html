Computer Networks

    Data Link Layer

    Introduction to Data Link Layer

    The Data Link Layer is the second layer in the OSI model and is responsible for node-to-node communication over a physical layer. It ensures reliable data transfer between adjacent network nodes.

    Key Responsibilities

    • Framing of network layer packets
    • Physical addressing (MAC addresses)
    • Error detection and control
    • Flow control
    • Access control to shared media

    Sublayers

    • LLC (Logical Link Control): Interface to network layer, flow control
    • MAC (Media Access Control): Controls hardware addressing and access to media

    IEEE 802 standards separate the Data Link Layer into these two sublayers

    Framing in Data Link Layer

    Framing is the process of encapsulating network layer packets into frames for transmission across the physical medium.

    Fixed-Size Framing

    Used in ATM networks where all frames are of identical size (53 bytes).

    [Header][Payload][Trailer] (all fixed size)

    Variable-Size Framing

    Used in Ethernet and most other networks where frame size varies.

    [Start Flag][Header][Variable Payload][Trailer][End Flag]

    Common Framing Methods

    Character Count

    Uses a field in header to indicate frame size. Vulnerable to count errors.

    Flag Bytes with Byte Stuffing

    Special flag bytes mark start/end. Escape characters used if flag appears in data.

    Bit Stuffing

    Special bit pattern (01111110) flags frame boundaries. Extra bit inserted if pattern appears in data.

    Physical Layer Coding Violations

    Uses invalid physical layer codes to indicate frame boundaries (used in some LANs).

    Error Detection and Correction

    Parity Checking

    • Single parity: One parity bit per byte (detects odd errors)
    • Two-dimensional parity: Parity bits for rows and columns (detects and corrects single-bit errors)

    Checksums

    • Sum of all data words is stored in checksum field
    • Receiver recomputes and compares checksums
    • Simple but not very robust

    CRC (Cyclic Redundancy Check)

    • Uses polynomial division to generate check bits
    • Detects all burst errors less than r+1 bits (r = degree of polynomial)
    • Standard polynomials: CRC-8, CRC-16, CRC-32

    Error Correction Techniques

    Hamming Codes

    Can detect and correct single-bit errors using parity bits at power-of-2 positions.

    Reed-Solomon Codes

    Block-based error correction that can handle burst errors, used in CDs/DVDs.

    Forward Error Correction (FEC)

    Adds redundant data to allow receiver to correct errors without retransmission.

    MAC (Media Access Control)

    MAC Addressing

    • 48-bit (6-byte) hardware addresses
    • First 3 bytes: OUI (Organizationally Unique Identifier)
    • Last 3 bytes: NIC specific
    • Written in hex (e.g., 00:1A:2B:3C:4D:5E)
    • Broadcast address: FF:FF:FF:FF:FF:FF

    MAC Protocols

    Methods to control access to shared media:

    • Channel Partitioning: TDMA, FDMA, CDMA
    • Random Access: ALOHA, CSMA, CSMA/CD, CSMA/CA
    • Taking Turns: Token passing, polling

    Common MAC Protocols Explained

    CSMA/CD (Ethernet)

    • Carrier Sense Multiple Access with Collision Detection
    • Listens before transmitting
    • Detects collisions and backs off
    • Used in wired Ethernet

    CSMA/CA (Wi-Fi)

    • Carrier Sense Multiple Access with Collision Avoidance
    • Uses RTS/CTS handshake
    • Includes random backoff time
    • Used in wireless networks

    Token Passing

    • Special token circulates in network
    • Only token holder can transmit
    • Used in Token Ring and FDDI networks
    • Deterministic access (no collisions)

    Data Link Layer Protocols

    Protocol Description Used In
    Ethernet (IEEE 802.3) Most common LAN technology using CSMA/CD Wired LANs
    Wi-Fi (IEEE 802.11) Wireless LAN standard using CSMA/CA Wireless networks
    PPP (Point-to-Point) Direct connection between two nodes Dial-up, leased lines
    HDLC Bit-oriented synchronous protocol Cisco routers, telecom
    Frame Relay Packet-switched WAN protocol WAN connections
    ATM Cell-based switching technology Telecom networks

    Switching at Data Link Layer

    Switching Concepts

    • Switches operate at Data Link Layer (Layer 2)
    • Maintain MAC address tables for forwarding decisions
    • Use store-and-forward or cut-through switching
    • Support VLANs for network segmentation

    Switch Functions

    • Address learning (builds MAC table)
    • Forward/filter decisions
    • Loop avoidance (using Spanning Tree Protocol)
    • VLAN support
    • Link aggregation

    Spanning Tree Protocol (STP)

    Prevents switching loops in redundant networks while maintaining backup paths:

    • Elects root bridge (switch with lowest bridge ID)
    • Calculates shortest path to root for each switch
    • Blocks redundant paths that could cause loops
    • Uses BPDUs (Bridge Protocol Data Units) for communication
    • Modern versions: RSTP (Rapid STP), MSTP (Multiple STP)
    • Convergence time is critical for network stability
