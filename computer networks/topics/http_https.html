    HTTP (Hypertext Transfer Protocol)

    Overview

    HTTP is the foundation of data communication for the World Wide Web. It is an application-layer protocol for transmitting hypermedia documents.

    Key Characteristics

    • Client-Server Model: HTTP follows a request-response paradigm between clients (browsers) and servers
    • Stateless: Each request is independent with no inherent memory of previous requests
    • TCP/IP Based: Typically operates over TCP/IP connections (usually port 80 for HTTP, 443 for HTTPS)
    • Text-Based: Human-readable format (though HTTP/2 introduced binary framing)

    HTTP Versions

    Version Year Key Features
    HTTP/0.9 1991 Single-line protocol, only GET method
    HTTP/1.0 1996 Headers, status codes, multiple methods
    HTTP/1.1 1997 Persistent connections, chunked transfers, host headers
    HTTP/2 2015 Binary protocol, multiplexing, header compression
    HTTP/3 2022 Uses QUIC (over UDP), improved security, faster handshakes

    HTTP Request Methods

    GET

    Requests a representation of the specified resource (should only retrieve data)

    POST

    Submits an entity to the specified resource, often causing state change

    PUT

    Replaces all current representations of the target resource

    DELETE

    Deletes the specified resource

    PATCH

    Applies partial modifications to a resource

    HEAD

    Same as GET but returns only headers, no body

    HTTP Status Codes

    1xx - Informational

    100 Continue, 101 Switching Protocols, 102 Processing, etc.

    2xx - Success

    200 OK, 201 Created, 204 No Content, 206 Partial Content, etc.

    3xx - Redirection

    301 Moved Permanently, 302 Found, 304 Not Modified, 307 Temporary Redirect, etc.

    4xx - Client Errors

    400 Bad Request, 401 Unauthorized, 403 Forbidden, 404 Not Found, 429 Too Many Requests, etc.

    5xx - Server Errors

    500 Internal Server Error, 501 Not Implemented, 502 Bad Gateway, 503 Service Unavailable, etc.

    HTTP Message Structure

    Request Message

                      GET /index.html HTTP/1.1
                  Host: www.example.com
                  User-Agent: Mozilla/5.0
                  Accept: text/html
                  Accept-Language: en-US
                  Connection: keep-alive

    Response Message

                      HTTP/1.1 200 OK
                  Date: Mon, 23 May 2023 22:38:34 GMT
                  Server: Apache/2.4.1
                  Last-Modified: Wed, 08 Jan 2023 23:11:55 GMT
                  Content-Type: text/html; charset=UTF-8
                  Content-Length: 138
                  
                  <html>
                  <head>
                    <title>Example Page</title>
                  </head>
                  <body>
                    <p>Hello World!</p>
                  </body>
                  </html>

    HTTPS (HTTP Secure)

    HTTPS is the secure version of HTTP that uses TLS/SSL encryption to protect data in transit:

    • Encrypts all communication between client and server
    • Verifies website authenticity
    • Protects against eavesdropping and man-in-the-middle attacks
    • Uses port 443 by default
    • Requires SSL/TLS certificate from a Certificate Authority

    Common HTTP APIs

    REST (Representational State Transfer)

    Architectural style using HTTP methods to perform CRUD operations on resources

    SOAP (Simple Object Access Protocol)

    XML-based protocol for exchanging structured information in web services

    GraphQL

    Query language for APIs that allows clients to request exactly what they need

    Performance Considerations

    • Persistent Connections: Reuse TCP connections for multiple requests
    • Compression: Use gzip or deflate to reduce payload size
    • Caching: Leverage browser and proxy caching with proper headers
    • CDN Usage: Distribute content geographically closer to users
    • HTTP/2: Adopt for multiplexing and header compression
