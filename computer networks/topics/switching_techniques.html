    Switching Techniques in Computer Networks

    Introduction to Switching

    Switching is the process of forwarding data packets from one network segment to another through intermediate network nodes called switches. Switching techniques determine how data is routed through a network.

    Purpose of Switching

    • Efficient data transmission
    • Optimal path selection
    • Traffic management
    • Network resource utilization

    Key Components

    • Switches (Layer 2/Layer 3)
    • Routing tables
    • Forwarding mechanisms
    • Network interfaces

    Types of Switching Techniques

    Technique Layer Description Use Cases
    Circuit Switching Physical Dedicated path established for entire communication Traditional telephone networks
    Packet Switching Network Data divided into packets that travel independently Internet, modern networks
    Message Switching Application Entire message is treated as a single unit Email systems (historical)
    Virtual Circuit Network Pre-established path for packet sequence ATM, Frame Relay
    Datagram Network Each packet routed independently IP networks

    1. Circuit Switching

    How It Works

    1. Establishment: Dedicated path created before communication
    2. Data Transfer: Continuous communication along the path
    3. Teardown: Connection terminated after completion

    Characteristics

    • Fixed bandwidth allocation
    • No sharing during connection
    • Predictable performance
    • Inefficient for bursty traffic

    Advantages

    • Guaranteed bandwidth
    • Low and constant latency
    • Simple management
    • No packet reordering needed

    Disadvantages

    • Inefficient resource utilization
    • High setup time
    • No fault tolerance
    • Costly for long-distance

    Applications

    PSTN

    Public Switched Telephone Network

    ISDN

    Integrated Services Digital Network

    Leased Lines

    Dedicated communication channels

    2. Packet Switching

    How It Works

    1. Packetization: Data divided into small packets
    2. Routing: Each packet routed independently
    3. Reassembly: Packets reassembled at destination

    Characteristics

    • Efficient bandwidth utilization
    • Store-and-forward mechanism
    • Variable latency
    • Supports multiple traffic types

    Advantages

    • Efficient resource sharing
    • Robust to failures
    • Supports variable data rates
    • Cost-effective

    Disadvantages

    • Variable latency (jitter)
    • Packet reordering possible
    • Overhead from headers
    • Congestion management needed

    Datagram Approach

    • Each packet treated independently
    • No connection setup required
    • Packets may take different paths
    • Used in IP networks

    Virtual Circuit Approach

    • Pre-established path for all packets
    • Connection-oriented
    • Packets follow same path
    • Used in ATM, Frame Relay

    Applications

    Internet

    IP-based communication

    LANs

    Ethernet networks

    VPNs

    Virtual Private Networks

    3. Message Switching

    How It Works

    1. Storage: Entire message stored at each node
    2. Forwarding: Message forwarded when next node available
    3. Delivery: Message delivered to destination

    Characteristics

    • Store-and-forward mechanism
    • No dedicated path
    • Variable delay
    • Supports large messages

    Advantages

    • Efficient use of network
    • Supports priority handling
    • No congestion during transfer
    • Error checking at each node

    Disadvantages

    • High latency
    • Requires large storage
    • Not suitable for real-time
    • Largely obsolete

    Historical Applications

    Telex

    Early teleprinter networks

    Email

    Early store-and-forward systems

    Comparison of Switching Techniques

    Feature Circuit Switching Packet Switching Message Switching
    Path Establishment Required before communication Not required (datagram) or required (virtual circuit) Not required
    Bandwidth Usage Dedicated, fixed Shared, dynamic Shared, dynamic
    Delay Characteristics Low, constant Variable (depends on traffic) High, variable
    Data Unit Continuous stream Packets Complete messages
    Fault Tolerance Low (connection breaks) High (rerouting possible) Moderate
    Modern Usage Telephone networks Internet, LANs Mostly obsolete
