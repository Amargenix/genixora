    Virtual Private Networks (VPNs)

    Introduction to VPN

    A Virtual Private Network (VPN) extends a private network across a public network, enabling users to send and receive data across shared or public networks as if their computing devices were directly connected to the private network.

    Key Characteristics

    • Creates secure "tunnel" over public internet
    • Encrypts all transmitted data
    • Hides user's real IP address
    • Provides access to restricted resources
    • Works across different devices and platforms

    Primary Benefits

    • Enhanced security and privacy
    • Bypass geographic restrictions
    • Secure remote access to company networks
    • Protection on public Wi-Fi networks
    • Prevent ISP tracking and throttling

    How VPNs Work

    VPN Connection Process:
    1. User initiates connection to VPN server
    2. VPN client encrypts all outgoing data
    3. Encrypted data travels through public internet
    4. VPN server decrypts the data
    5. Server forwards request to destination (web server, etc.)
    6. Response travels back through the encrypted tunnel

    VPN Encryption Process

    1. Handshake Phase

    • Client and server authenticate each other
    • Negotiate encryption protocols
    • Establish secure session keys

    2. Tunnel Establishment

    • Create encrypted communication channel
    • Set up data encapsulation protocol
    • Verify connection integrity

    3. Data Transmission

    • All data encrypted before transmission
    • Packets encapsulated with VPN headers
    • Secure transmission through public network

    4. Termination

    • Secure session closure
    • Keys discarded
    • Connection logs updated

    Types of VPNs

    1. Remote Access VPN

    Allows individual users to connect to a private network from remote locations.

    Use Cases:

    • Employees working from home
    • Business travelers accessing company resources
    • Secure access to files and applications

    Examples: Corporate VPNs, Commercial VPN services

    2. Site-to-Site VPN

    Connects entire networks to each other, typically used by businesses with multiple offices.

    Use Cases:

    • Connecting branch offices to headquarters
    • Linking data centers
    • Secure cloud connectivity

    Examples: MPLS VPNs, IPsec VPN tunnels

    3. Client-to-Site VPN

    Similar to remote access VPN but typically refers to individual devices connecting to a network.

    Use Cases:

    • Contractors accessing specific resources
    • Temporary employee access
    • Secure BYOD (Bring Your Own Device) policies

    4. SSL/TLS VPN

    Provides secure remote access through a web browser without requiring dedicated client software.

    Use Cases:

    • Access from public computers
    • Quick access to web applications
    • When client installation isn't possible

    Examples: Citrix Gateway, OpenVPN Access Server

    VPN Security Features

    Encryption

    Scrambles data to prevent interception

    Algorithms: AES, Blowfish, Camellia

    Tunneling

    Encapsulates data packets within other packets

    Methods: PPTP, L2TP, IPsec

    Authentication

    Verifies user identity before granting access

    Methods: Certificates, 2FA, OTP

    Kill Switch

    Blocks internet if VPN connection drops

    Prevents data leaks

    DNS Leak Protection

    Ensures DNS queries go through VPN

    Prevents exposure of browsing activity

    Split Tunneling

    Routes some traffic through VPN while other traffic accesses internet directly
