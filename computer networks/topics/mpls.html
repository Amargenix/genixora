    MPLS in Computer Networks

    Introduction to MPLS

    What is MPLS?

    MPLS (Multiprotocol Label Switching) is a packet-forwarding technology that uses labels to make data forwarding decisions. It operates between Layer 2 (Data Link) and Layer 3 (Network) in the OSI model, often called "Layer 2.5".

    Key Characteristics:
    • Uses label switching instead of IP routing
    • Creates virtual paths (Label Switched Paths - LSPs)
    • Supports multiple protocols (IPv4, IPv6, Ethernet, ATM, Frame Relay)
    • Enables Traffic Engineering (TE)
    • Provides Quality of Service (QoS) capabilities

    MPLS vs Traditional IP Routing

    Forwarding Label-based Destination-based

    MPLS Architecture

    MPLS Network Components

    • Label Edge Router (LER): Ingress/Egress routers that add/remove labels
    • Label Switching Router (LSR): Core routers that switch packets based on labels
    • Label Switched Path (LSP): The predetermined path through the MPLS network
    • Forwarding Equivalence Class (FEC): Group of packets treated the same way

    MPLS Label Stack

    MPLS uses a 32-bit label header inserted between Layer 2 and Layer 3 headers:

    20 bits
    Label    		 3 bits
    Traffic Class (QoS)    		 1 bit
    Bottom of Stack    		 8 bits
    TTL

    Multiple labels can be stacked (used in MPLS VPNs and traffic engineering).

    MPLS Operation

    1. Label Assignment: LER assigns labels to incoming packets based on FEC
    2. Label Distribution: LSRs exchange label information using LDP, RSVP-TE, or BGP
    3. Label Switching: LSRs forward packets based on label lookups
    4. Label Removal: Egress LER removes the label and forwards natively
    Label Operations:
    • Push: Add a new label (at ingress)
    • Swap: Replace label with new one (core LSRs)
    • Pop: Remove label (penultimate hop or egress)

    Penultimate Hop Popping (PHP)

    The next-to-last LSR removes the label to reduce processing on the egress LER. The egress LER signals this using an implicit-null label (value 3).

    MPLS Protocols

    Label Distribution Protocols

    LDP (Label Distribution Protocol)

    Standard protocol for basic MPLS label distribution (RFC 5036). Uses TCP port 646.

    • Distributes labels based on existing IP routing
    • Uses Hello messages (UDP port 646) for neighbor discovery
    • Four message types: Discovery, Session, Advertisement, Notification

    RSVP-TE (Resource Reservation Protocol - Traffic Engineering)

    Used for traffic engineering and establishing explicit paths (RFC 3209).

    • Supports bandwidth reservations
    • Enables constraint-based routing
    • Provides fast reroute capabilities

    MP-BGP (Multiprotocol BGP)

    Extended version of BGP used for MPLS VPNs (RFC 4760).

    • Carries VPN routing information between PE routers
    • Uses Route Distinguishers (RD) and Route Targets (RT)
    • Supports IPv4, IPv6, and other address families

    MPLS Applications

    MPLS VPNs

    MPLS enables Layer 3 VPN services where customer routes are kept separate using VRFs (Virtual Routing and Forwarding).

    Components:

    • CE (Customer Edge): Customer router connecting to provider
    • PE (Provider Edge): Provider router maintaining VRFs
    • P (Provider): Core MPLS routers

    How it Works:

    1. PE routers maintain separate VRFs for each customer
    2. Customer routes are exchanged via MP-BGP
    3. Two labels are used: outer label (transport) and inner label (VPN)

    MPLS Traffic Engineering

    Allows network operators to control the path traffic takes through the network.

    • Override shortest-path routing
    • Balance network utilization
    • Meet QoS requirements
    • Provide fast reroute capabilities

    Other MPLS Applications

    • MPLS-TP: Transport profile for carrier networks
    • MPLS QoS: Class of Service (CoS) markings
    • MPLS Multicast: Multicast VPN (mVPN)
    • Ethernet over MPLS: EoMPLS (Pseudowires)
    • Segment Routing: Modern evolution of MPLS

    MPLS Benefits and Challenges

    Advantages of MPLS

    • Faster forwarding: Label lookups are simpler than IP routing
    • Traffic engineering: Control over network paths
    • QoS support: Built-in traffic prioritization
    • VPN services: Scalable Layer 3 VPNs
    • Convergence: Fast reroute capabilities
    • Any transport: Works over various Layer 2 technologies

    Challenges and Considerations

    • Complexity: Requires additional protocols and configuration
    • Cost: MPLS-capable equipment is expensive
    • Scalability: Label space management in large networks
    • Visibility: Harder to troubleshoot than IP networks
    • Competition: SD-WAN and other technologies emerging
    Future of MPLS: Evolving with Segment Routing (SR-MPLS) which simplifies operation by removing LDP and RSVP-TE dependencies.
