    Cloud Infrastructure in Computer Networks

    Introduction to Cloud Infrastructure

    Cloud infrastructure refers to the hardware and software components needed for cloud computing, including servers, storage, networking, virtualization software, services, and management tools. It enables on-demand access to shared computing resources.

    Key Characteristics

    • On-demand self-service
    • Broad network access
    • Resource pooling
    • Rapid elasticity
    • Measured service

    Benefits

    • Cost efficiency
    • Scalability
    • Flexibility
    • Reliability
    • Disaster recovery

    Cloud Service Models

    Infrastructure as a Service (IaaS)

    Provides fundamental computing resources:

    • Virtual machines
    • Storage
    • Networking
    • Operating systems

    Examples: AWS EC2, Azure VMs, Google Compute Engine

    Platform as a Service (PaaS)

    Provides development environments:

    • Runtime environments
    • Development tools
    • Middleware
    • Database management

    Examples: AWS Elastic Beanstalk, Azure App Service

    Software as a Service (SaaS)

    Provides complete applications:

    • Web-based email
    • Office productivity suites
    • CRM systems

    Examples: Gmail, Office 365, Salesforce

    Cloud Deployment Models

    Model Description Use Cases
    Public Cloud Resources owned by cloud provider, available to general public Web applications, SaaS, development environments
    Private Cloud Resources used exclusively by single organization Financial data, healthcare systems, government applications
    Hybrid Cloud Combination of public and private clouds Bursting, disaster recovery, legacy systems
    Multi-Cloud Use of multiple cloud providers Vendor lock-in avoidance, specialized services

    Cloud Networking Components

    1. Virtual Networks

    Software-defined networks that connect cloud resources:

    • VPC (Virtual Private Cloud): Isolated network for cloud resources
    • Subnets: Segments within VPCs for different purposes
    • CIDR Blocks: IP address ranges for network design

    2. Connectivity Options

    Internet Gateway

    Horizontally scaled, redundant component for internet communication

    VPN Connections

    Secure encrypted tunnels between cloud and on-premises

    Direct Connect

    Dedicated network connections bypassing the internet

    Cloud Security Infrastructure

    Identity and Access Management (IAM)

    • User authentication
    • Role-based access control
    • Multi-factor authentication

    Network Security

    • Security groups
    • Network ACLs
    • Web Application Firewalls

    Data Protection

    • Encryption at rest and in transit
    • Key management services
    • Data loss prevention

    Cloud Storage Infrastructure

    Storage Type Description Use Cases
    Object Storage Unstructured data storage with metadata Backups, media files, static websites
    Block Storage Raw storage volumes for compute instances Databases, enterprise applications
    File Storage Network-attached file systems Content management, shared access
    Archive Storage Low-cost storage for rarely accessed data Compliance archives, long-term backups

    Emerging Cloud Technologies

    Serverless Computing

    Execution model where cloud provider runs server and manages allocation

    Examples: AWS Lambda, Azure Functions

    Edge Computing

    Distributed computing bringing computation closer to data sources

    Examples: AWS Outposts, Azure Stack Edge

    Hybrid Cloud Management

    Tools for managing resources across multiple environments

    Examples: Azure Arc, Google Anthos

    Cloud Migration Strategies

    Rehost (Lift and Shift)

    Moving applications without modification

    Best for: Quick migrations, legacy apps

    Refactor (Lift and Reshape)

    Minor optimizations for cloud

    Best for: Apps needing cloud features

    Rearchitect

    Significant modification for cloud-native

    Best for: Long-term cloud benefits
