    Transport Layer

    Layer 4 of the OSI Model - End-to-End Communication Services

    Introduction to Transport Layer

    The Transport Layer (Layer 4) provides end-to-end communication services for applications running on different hosts. It's a critical layer that ensures data is delivered reliably and efficiently across networks.

    Key Responsibilities

    • End-to-end message delivery
    • Error control and flow control
    • Congestion control
    • Multiplexing and demultiplexing
    • Connection establishment and termination

    Position in Network Stack

    • OSI Model: Layer 4
    • TCP/IP Model: Transport Layer
    • Sits above Network Layer (IP)
    • Below Session/Application Layers
    • Bridges network and application layers

    Transport Layer Protocols

    TCP (Transmission Control Protocol)

    • Connection-oriented protocol
    • Provides reliable, ordered delivery
    • Implements flow and congestion control
    • Uses acknowledgments and retransmissions
    • Three-way handshake for connection establishment
    • Header size: 20-60 bytes

    UDP (User Datagram Protocol)

    • Connectionless protocol
    • Unreliable but faster delivery
    • No flow or congestion control
    • No connection establishment
    • Minimal overhead (8-byte header)
    • Used for real-time applications

    Other Transport Protocols

    SCTP

    Stream Control Transmission Protocol - combines features of TCP and UDP with multi-homing support

    DCCP

    Datagram Congestion Control Protocol - provides congestion control for unreliable flows

    QUIC

    Quick UDP Internet Connections - Google's protocol combining UDP with reliability features

    Key Functions of Transport Layer

    1. Multiplexing and Demultiplexing

    Allows multiple applications to use network services simultaneously using port numbers.

    • Port numbers identify applications (HTTP=80, FTP=21, etc.)
    • Well-known ports: 0-1023
    • Registered ports: 1024-49151
    • Dynamic/private ports: 49152-65535

    2. Connection Management

    TCP establishes, maintains, and terminates connections between hosts.

    • Three-way handshake: SYN ‚Üí SYN-ACK ‚Üí ACK
    • Connection termination: FIN ‚Üí ACK ‚Üí FIN ‚Üí ACK
    • Sequence numbers track data segments
    • Sliding window protocol manages flow

    3. Reliable Data Transfer

    Ensures data arrives correctly and in order (TCP only).

    • Sequence numbers for ordering
    • Acknowledgments for received data
    • Retransmission of lost packets
    • Checksums for error detection

    4. Flow and Congestion Control

    Manages data flow to prevent overwhelming receivers or networks.

    • Flow control: Prevents receiver overflow
    • Congestion control: Prevents network congestion
    • TCP uses window-based mechanisms
    • Algorithms: Slow Start, AIMD, Fast Retransmit

    TCP vs UDP Comparison

    Feature TCP UDP
    Connection Connection-oriented Connectionless
    Reliability Reliable (acknowledgments) Unreliable
    Ordering Sequenced delivery No sequencing
    Speed Slower (more overhead) Faster (less overhead)
    Flow Control Yes (sliding window) No flow control
    Congestion Control Yes (multiple algorithms) No congestion control
    Header Size 20-60 bytes 8 bytes (fixed)
    Use Cases Web, email, file transfer Video streaming, VoIP, DNS
