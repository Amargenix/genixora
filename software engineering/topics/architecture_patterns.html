Software Engineering

    Software Architecture & Design Patterns

    1. Software Architecture Fundamentals

    Software architecture defines the high-level structure of a system and how components interact.

    Key Architectural Principles

    • Separation of Concerns: Divide system into distinct features
    • Single Responsibility: Each component does one thing well
    • Loose Coupling: Minimize dependencies between components
    • High Cohesion: Related functionality stays together

    Common Architectural Styles

    Style Description When to Use Example
    Monolithic Single unified codebase Small applications, simple projects Basic CMS systems
    Layered (N-tier) Presentation → Business Logic → Data layers Enterprise applications Banking software
    Microservices Independent services communicating via APIs Large, complex systems needing scalability Netflix, Uber
    Event-Driven Components communicate through events Real-time systems Stock trading platforms
    Serverless Cloud-managed functions (FaaS) Short-lived, event-based tasks Image processing, notifications

    2. Design Patterns

    Reusable solutions to common software design problems.

    A. Creational Patterns (Object Creation)

    Singleton Pattern

    Purpose: Ensure a class has only one instance and provide global access

    Use Case: Database connections, logging systems

    Example Implementation (Java):

    public class Database { private static Database instance; private Database() {} // Private constructor public static Database getInstance() { if (instance == null) { instance = new Database(); } return instance; } }

    Factory Method Pattern

    Purpose: Create objects without specifying exact class

    Use Case: Payment gateways, document converters

    Example Implementation (Python):

    from abc import ABC, abstractmethod class Payment(ABC): @abstractmethod def pay(self, amount): pass class CreditCard(Payment): def pay(self, amount): print(f"Paid {amount} via Credit Card") class PayPal(Payment): def pay(self, amount): print(f"Paid {amount} via PayPal") class PaymentFactory: def create_payment(self, method): if method == "credit": return CreditCard() elif method == "paypal": return PayPal() else: raise ValueError("Unknown payment method")

    B. Structural Patterns (Object Composition)

    Adapter Pattern

    Purpose: Make incompatible interfaces work together

    Use Case: Legacy system integration, third-party APIs

    Example Scenario:

    Adapting a European plug (client) to work with a US socket (adaptee)

    Decorator Pattern

    Purpose: Add responsibilities to objects dynamically

    Use Case: Input/output streams, middleware

    Example Implementation (JavaScript):

    class Coffee { cost() { return 5; } } class MilkDecorator { constructor(coffee) { this.coffee = coffee; } cost() { return this.coffee.cost() + 2; } } let myCoffee = new Coffee(); myCoffee = new MilkDecorator(myCoffee); console.log(myCoffee.cost()); // Output: 7

    C. Behavioral Patterns (Object Interaction)

    Observer Pattern

    Purpose: Define one-to-many dependency between objects

    Use Case: Event handling, news feeds

    Example Implementation (C#):

    public interface IObserver { void Update(string message); } public class NewsAgency { private List observers = new List(); public void AddObserver(IObserver observer) { observers.Add(observer); } public void Notify(string news) { foreach (var observer in observers) { observer.Update(news); } } }

    Strategy Pattern

    Purpose: Encapsulate interchangeable algorithms

    Use Case: Sorting algorithms, payment methods

    Example Implementation (Python):

    from abc import ABC, abstractmethod class CompressionStrategy(ABC): @abstractmethod def compress(self, file): pass class ZipCompression(CompressionStrategy): def compress(self, file): print(f"Compressing {file} using ZIP") class RarCompression(CompressionStrategy): def compress(self, file): print(f"Compressing {file} using RAR") class Compressor: def __init__(self, strategy): self.strategy = strategy def compress_file(self, file): self.strategy.compress(file)

    3. Architecture vs Design Patterns

    Aspect Software Architecture Design Patterns
    Scope System-wide (macro level) Component/class level (micro level)
    Purpose Define system structure and components Solve specific design problems
    Abstraction High-level (boxes and lines) Lower-level (class relationships)
    Examples Microservices, MVC, Event-Driven Singleton, Observer, Factory
