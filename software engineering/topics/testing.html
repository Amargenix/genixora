Software Engineering

    Software Testing and Quality Assurance

    1. Testing Fundamentals

    1.1 What is Software Testing?

    Software testing is the process of evaluating a software application to:

    • Identify defects and verify requirements are met
    • Ensure the software is fit for its intended purpose
    • Improve quality by finding bugs before release
    • Reduce risk of failures in production

    Key Testing Principles

    • Testing shows presence of defects: Can't prove absence
    • Exhaustive testing is impossible: Need smart test cases
    • Early testing: Start testing requirements
    • Defect clustering: Small modules often have most defects
    • Pesticide paradox: Tests need regular review

    1.2 Quality Assurance vs Quality Control

    Aspect Quality Assurance (QA) Quality Control (QC)
    Focus Process-oriented (prevent defects) Product-oriented (identify defects)
    When Throughout development lifecycle During/after development
    Activities Process definition, audits, training Testing, inspection, review
    Goal Improve processes to prevent issues Find defects in the product

    2. Types of Software Testing

    2.1 Functional Testing

    Unit Testing

    • Tests individual components/modules
    • Performed by developers
    • Tools: JUnit, NUnit, pytest

    Integration Testing

    • Tests interaction between components
    • Approaches: Big Bang, Top-Down, Bottom-Up
    • Tools: TestNG, Postman (for APIs)

    System Testing

    • End-to-end testing of complete system
    • Validates against requirements
    • Includes functional and non-functional tests

    User Acceptance Testing (UAT)

    • Final validation by business/users
    • Types: Alpha, Beta, Contract Acceptance
    • Focuses on business workflows

    2.2 Non-Functional Testing

    Performance Testing

    • Load Testing: System behavior under expected load
    • Stress Testing: Beyond normal operational capacity
    • Endurance Testing: Long duration under expected load
    • Tools: JMeter, LoadRunner, Gatling

    Security Testing

    • Identifies vulnerabilities and threats
    • Types: Penetration testing, SAST, DAST
    • Tools: OWASP ZAP, Burp Suite, Nessus

    Usability Testing

    • Evaluates user-friendliness
    • Measures learnability, efficiency, satisfaction
    • Methods: Heuristic evaluation, user sessions

    Compatibility Testing

    • Tests across browsers, devices, OS versions
    • Tools: BrowserStack, Sauce Labs

    2.3 Testing Techniques

    Black Box Testing

    • Tests functionality without code knowledge
    • Techniques: Equivalence Partitioning, Boundary Value Analysis
    • Performed by testers

    White Box Testing

    • Tests internal structures/code
    • Techniques: Statement Coverage, Branch Coverage
    • Performed by developers

    Grey Box Testing

    • Combination of black and white box
    • Partial knowledge of internal workings
    • Used for integration testing

    3. Testing Process and Methodologies

    3.1 Test Planning

    Test Plan Components

    1. Objectives: What needs to be tested
    2. Scope: Features to be tested/not tested
    3. Approach: Testing techniques and types
    4. Schedule: Timeline and milestones
    5. Resources: People, tools, environments
    6. Deliverables: Test cases, reports, metrics

    3.2 Test Case Development

    Characteristics of Good Test Cases

    • Atomic: Tests one specific thing
    • Repeatable: Same results every time
    • Traceable: Linked to requirements
    • Self-verifying: Clear pass/fail criteria

    Sample Test Case Format

    Test Case ID: TC_LOGIN_001
    Description: Verify successful login with valid credentials
    Preconditions: User account exists
    Test Steps:
    1. Navigate to login page
    2. Enter valid username
    3. Enter valid password
    4. Click Login button
    Expected Result: User is redirected to dashboard

    3.3 Defect Lifecycle

    Stages of a Defect

    1. New: Initial logging of defect
    2. Assigned: Given to developer
    3. Open: Developer working on fix
    4. Fixed: Developer completes fix
    5. Retest: Tester verifies fix
    6. Verified: Fix confirmed
    7. Closed: Defect resolved

    Effective Bug Reporting

    • Clear, concise title
    • Detailed reproduction steps
    • Expected vs actual results
    • Environment details
    • Screenshots/logs when applicable

    4. Test Automation

    4.1 Test Automation Strategy

    What to Automate?

    • Repetitive test cases
    • Data-driven tests
    • Regression test suites
    • Performance/load tests
    • Complex calculations

    Automation vs Manual Testing

    Factor Automation Manual
    Execution Speed Fast Slow
    Initial Investment High Low
    Human Judgment Limited Excellent
    Maintenance Required Minimal

    4.2 Test Automation Frameworks

    Popular Automation Tools

    • Web: Selenium, Cypress, Playwright
    • API: Postman, RestAssured, SoapUI
    • Mobile: Appium, Espresso, XCUITest
    • Performance: JMeter, LoadRunner
    • Unit Testing: JUnit, TestNG, pytest

    Framework Types

    • Linear: Simple record and playback
    • Modular: Test scripts divided by functionality
    • Data-Driven: Tests run with multiple data sets
    • Keyword-Driven: Tests described with keywords
    • Hybrid: Combines multiple approaches

    5. Quality Metrics and Reporting

    5.1 Key Testing Metrics

    Metric Purpose Formula
    Test Coverage Requirements/test cases covered (Tested Requirements/Total Requirements) √ó 100
    Defect Density Quality of software modules Defects Found/Size (KLOC or FP)
    Defect Removal Efficiency Effectiveness of testing (Defects Found During Testing/Total Defects) √ó 100
    Test Case Effectiveness Quality of test cases (Defects Found by Test Case/Total Test Cases) √ó 100

    5.2 Test Reporting

    Essential Test Reports

    • Test Summary Report: Overall testing results
    • Defect Report: Analysis of found defects
    • Traceability Matrix: Requirements vs test cases
    • Test Progress Report: Current status vs plan

    6. Emerging Trends in Testing

    6.1 Modern Testing Approaches

    • AI in Testing: Self-healing tests, visual validation
    • Shift-Left Testing: Early testing in SDLC
    • Shift-Right Testing: Production monitoring
    • Continuous Testing: CI/CD pipeline integration
    • Service Virtualization: Simulate unavailable systems

    6.2 Future of QA

    • Increased test automation coverage
    • More focus on security and performance
    • QA as quality coaches rather than gatekeepers
    • Greater emphasis on user experience testing
    • Integration of testing with DevOps (TestOps)
