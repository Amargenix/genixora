    2. Software Development Methodologies

    Waterfall

    • Linear sequential approach
    • Requirements ‚Üí Design ‚Üí Implementation ‚Üí Testing ‚Üí Maintenance
    • Best for: Stable, well-defined projects

    Agile

    • Iterative and incremental
    • 2-4 week sprints with continuous feedback
    • Best for: Dynamic requirements, fast-changing markets

    DevOps

    • Combines development and operations
    • CI/CD pipelines, infrastructure as code
    • Best for: Frequent releases, cloud-native apps

    Methodology Selection Criteria

    Factor Waterfall Agile DevOps
    Requirements Stability High Low-Medium Any
    Team Size Large Small-Medium Any
    Release Frequency Months-Years Weeks-Months Days-Weeks

    4. Language-Specific Standards

    JavaScript

    • Use const/let instead of var
    • Follow Airbnb or Google style guide
    • Use ES6+ features (arrow functions, templates)

    Python

    • Follow PEP 8 guidelines
    • 4-space indentation
    • Use snake_case for variables

    Java

    • CamelCase for methods/variables
    • PascalCase for classes
    • Follow Oracle Code Conventions
