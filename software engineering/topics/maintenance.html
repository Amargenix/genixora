    Software Maintenance & Evolution

    1. Introduction to Software Maintenance

    Why Maintenance Matters

    • 60-80% of total software costs are maintenance costs
    • Average system lifespan: 10-15 years
    • 70% of maintenance effort goes to enhancements

    2. Types of Software Maintenance

    Corrective Maintenance

    Fixing defects and errors

    // Example: Bug fix in authentication middleware function authenticateUser(user) { // Fixed null reference exception if (!user || !user.credentials) { throw new Error("Invalid user object"); } // ... existing logic }
    Type % Effort Purpose
    Corrective 20% Fix defects
    Adaptive 25% Environment changes
    Perfective 50% Enhancements
    Preventive 5% Future-proofing

    3. Software Evolution Principles

    Lehman's Laws of Software Evolution

    1

    Continuing Change

    Systems must be continually adapted or become progressively less satisfactory

    2

    Increasing Complexity

    As systems evolve, their complexity increases unless work is done to maintain/ reduce it
