    Software Licensing and Intellectual Property

    1. Intellectual Property Fundamentals

    1.1 Types of IP Protection for Software

    Protection Type Applies To Duration Key Characteristics
    Copyright Source code, documentation Author's life + 70 years (varies by country) Automatic protection, prevents copying
    Patents Innovative processes, algorithms 20 years from filing Requires application, protects ideas
    Trademarks Software names, logos Indefinite with renewal Protects brand identity
    Trade Secrets Proprietary algorithms, formulas As long as kept secret No registration, protected by NDAs

    1.2 Copyright vs. Patents for Software

    Feature Copyright Patent
    Protects Expression of ideas (code) Ideas and functionality
    Requirements Original work fixed in tangible medium Novel, non-obvious, useful invention
    Automatic Yes (upon creation) No (requires application)
    Cost Free (registration optional) $5,000-$15,000+
    Example Protects specific code implementation Protects novel compression algorithm

    2. Software Licensing Models

    2.1 Proprietary Software Licenses

    Commercial Proprietary

    • Restrictive usage terms
    • No source code access
    • Examples: Microsoft Office, Adobe Photoshop
    • Typical clauses: No reverse engineering, limited installations

    Freemium

    • Basic version free, premium features paid
    • Examples: Slack, Dropbox
    • Monetization: Upselling to paid plans

    2.2 Open Source Licenses

    Permissive Licenses

    • MIT License: Minimal restrictions, requires attribution
    • Apache 2.0: Includes patent grant
    • BSD: Similar to MIT with additional clauses
    • Examples: React (MIT), Android (Apache)

    Copyleft Licenses

    • GPL: Requires derivative works to be open source
    • LGPL: Less restrictive for libraries
    • AGPL: Extends GPL to network use
    • Examples: Linux (GPL), WordPress (GPL)

    2.3 Comparison of Common Open Source Licenses

    License Modification Redistribution Patent Use Commercial Use
    MIT Allowed Allowed with notice Implied Allowed
    Apache 2.0 Allowed Allowed with notice Explicit grant Allowed
    GPL v3 Allowed (must share changes) Allowed (must share source) Explicit grant Allowed (with source)

    3. Legal Considerations

    3.1 Key License Compliance Issues

    • Copyleft Contamination: Mixing GPL with proprietary code
    • Attribution Requirements: Proper license notices
    • Patent Risks: Apache/MIT vs GPL patent clauses
    • Export Controls: Encryption software restrictions

    Case Study: Oracle vs. Google (API Copyright)

    Issue: Whether APIs are copyrightable

    Outcome: Supreme Court ruled Google's use of Java APIs was fair use

    Impact: Clarified boundaries of software copyright protection

    3.2 Software Audits and Compliance

    1. Maintain a software bill of materials (SBOM)
    2. Track all open source components and their licenses
    3. Implement approval processes for new dependencies
    4. Conduct regular license compliance audits
    5. Train developers on license implications
